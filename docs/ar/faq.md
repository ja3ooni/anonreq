# الأسئلة الشائعة

## ماذا يحدث عندما يفشل الكشف عن PII؟

تفشل البوابة بأمان (fail-secure). في حالة حدوث أخطاء في الكشف أو الذاكرة المؤقتة أو مهلة الموفر، فإن الطلب يعود بـ HTTP 5xx ولا يتم توجيه أي بيانات إلى الموفرين الخارجيين. راجع بنية الأمان في ملف README الخاص بالمشروع للحصول على التفاصيل.

## هل يتم حفظ بياناتي في أي مكان؟

لا. يتم تخزين جميع تعيينات PII إلى الرموز في Valkey دون استمرار (`save ""`). يتم حذف التعيينات بعد إرسال الاستجابة. تحتوي السجلات على بيانات تعريف فقط — لا توجد قيم PII أولية.

## ما هي جهات توفير LLM المدعومة؟

OpenAI و Azure OpenAI و Anthropic (Claude) و Google Gemini و Ollama (الموديلات المحلية). تترجم البوابة تنسيق الطلب المتوافق مع OpenAI إلى البروتوكول الأصلي لكل موفر خدمة.

## كيف يعمل البث (streaming)؟

تستخدم البوابة FSM من نوع Tail_Buffer للتعامل مع الرموز المقسمة عبر حدود أجزاء SSE. يتم استعادة الرموز في الوقت الفعلي مع وصول الأجزاء. الاستجابة متطابقة بايت بايت مع وضع عدم البث.

## ما هو تنسيق الرمز (token format)؟

يتم استبدال PII المكتشفة بعلامات حجز `[TYPE_N]`، حيث يكون `TYPE` هو نوع الكيان (على سبيل المثال، `EMAIL` أو `PHONE`) و `N` هو فهرس فريد. لا يفرق مطابقة الرموز بين حالة الأحرف وتكون الأقواس اختيارية أثناء الاستعادة.

## كيف يتم التعامل مع الإعدادات الإقليمية (locales)؟

قم بتعيين ترويسة `X-AnonReq-Locale` لتنشيط الكشف المخصص للإعداد الإقليمي. يمكن دمج إعدادات إقليمية متعددة (مفصولة بفاصلة). تعيد الإعدادات الإقليمية غير المدعومة HTTP 400.

## هل يمكنني إضافة أنماط كشف مخصصة؟

نعم، يمكن إضافة قواعد regex المخصصة عبر ملف تكوين YAML وإعادة تحميلها بدون إعادة تشغيل. راجع وثائق التكوين لمعرفة تنسيق القواعد.

## كيف يمكنني المساهمة؟

المساهمات مرحب بها بموجب ترخيص Apache 2.0. راجع دليل المساهمة في المستودع للحصول على إرشادات طلب السحب وإعداد التطوير.
