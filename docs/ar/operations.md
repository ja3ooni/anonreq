# دليل العمليات لـ AnonReq

يوفر هذا الدليل مخططات تشغيلية وهياكل تكوين ومواصفات مراقبة وخطوات استكشاف الأخطاء وإصلاحها لمهندسي النظام الذين يديرون بوابة AnonReq.

## إدارة التكوين

يتم إدارة تكوينات البوابة عبر ملفات YAML التي يتم تحميلها عند بدء تشغيل الحاوية. التكوينات الرئيسية هي:

- **محرك السياسات (`config/policy.yaml`):** يحدد قواعد المستأجر وحدود معدل الطلب وميزانيات الإنفاق وحدود مكان إقامة البيانات.
- **تكوين SLO (`config/slo.yaml`):** يعلن عن الأهداف التشغيلية لمعدل النجاح، وزمن الانتقال، وحالات الأمان، وسجلات المراجعة.
- **بوابة Trust Center (`config/trust_center.yaml`):** تنظم الوصول إلى بوابة الامتثال العامة.

يتم إعادة تحميل التكوينات تلقائيًا عند تلقي إشارة `SIGHUP`.

### مثال لتكوين السياسات

```yaml
version: "1.0"
rules:
  - rule_id: "block_restricted_pii"
    name: "Block Restricted Data"
    action: "BLOCK"
    priority: 100
    enabled: true
    conditions:
      classification_level: "Restricted"
rate_limits:
  enabled: true
  rpm: 1000
```

## مراقبة أهداف مستوى الخدمة (SLOs)

يتتبع AnonReq 4 أهداف رئيسية لمستوى الخدمة لضمان السلامة التشغيلية وأداء البوابة:

1. **معدل النجاح:** يجب أن ينجح ما لا يقل عن 99.9% من طلبات البوابة.
2. **زمن الانتقال P95:** يجب أن يظل زمن الانتقال للمعالجة أقل من أو يساوي 100 مللي ثانية.
3. **معدل الأمان (Fail-Secure):** يجب ألا تؤدي أكثر من 0.1% من المعاملات إلى حظر الأمان.
4. **معدل كتابة المراجعة:** يجب إكمال ما لا يقل عن 99.99% من كتابات سجل المراجعة بنجاح.

### البنية التحتية للمراقبة

- **لوحة معلومات Prometheus:** تجمع المقاييس من `/metrics` على المنفذ `8080`.
- **لوحة معلومات Grafana:** تعرض حالة الامتثال لأهداف مستوى الخدمة.

## العمليات الإدارية عبر CLI

يستخدم مهندسو النظام طلبات curl للاستعلام عن الحالة وتحديث السياسات.

### 1. التحقق من السياسات النشطة
```bash
curl -X GET http://localhost:8080/v1/admin/policies   -H "Authorization: Bearer <ADMIN_API_KEY>"   -H "X-AnonReq-Role: operator"   -H "X-AnonReq-Tenant-ID: default"
```

### 2. الاستعلام عن الامتثال لأهداف مستوى الخدمة في الوقت الفعلي
```bash
curl -H "Authorization: Bearer <ADMIN_API_KEY>"      -H "X-AnonReq-Role: administrator"      http://localhost:8080/v1/governance/status
```

### 3. التحقق من سلامة سلسلة المراجعة المشفرة
```bash
curl -X POST http://localhost:8080/v1/governance/audit/verify   -H "Authorization: Bearer <ADMIN_API_KEY>"   -H "X-AnonReq-Role: administrator"
```

## استكشاف الأخطاء وإصلاحها والتعافي

عند انتهاك هدف مستوى الخدمة، تصدر البوابة تنبيهات تلقائيًا. يجب على المشغلين التحقق من الأنظمة الفرعية التالية:

- **انخفاض معدل النجاح:** تحقق من الاتصال بموفري الخدمة الخارجيين وافحص استهلاك موارد Valkey.
- **ارتفاع زمن الانتقال:** تحقق من استخدام موارد حاويات البوابة وقم بزيادة عدد النسخ المتماثلة إذا لزم الأمر.
- **زيادة معدل حظر الأمان:** افحص سجلات حاوية Presidio Analyzer للتحقق من استجابتها.
- **فشل كتابة المراجعة:** تحقق من اتصال Valkey أو سعة قاعدة بيانات SQL ومساحة القرص.
