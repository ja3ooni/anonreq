# Presets الخاصة بالامتثال

## نظرة عامة

يوفر AnonReq ميزات مسبقة التكوين للامتثال (compliance presets) تفرض الكشف عن الكيانات المفروضة لأطر تنظيمية محددة. ينشط كل preset مجموعة من التعرفات المطلوبة بموجب اللائحة المقابلة.

| الـ Preset | اللائحة | المنطقة | الكيانات المفروضة |
|--------|------------|--------|-------------------|
| `gdpr` | اللائحة العامة لحماية البيانات | الاتحاد الأوروبي | البريد الإلكتروني، الهاتف، الاسم، العنوان، IP، الهوية الوطنية |
| `lgpd` | Lei Geral de Proteção de Dados | البرازيل | CPF، CNPJ، البريد الإلكتروني، الهاتف، الاسم، العنوان |
| `pdpa` | قانون حماية البيانات الشخصية | تايلاند | البريد الإلكتروني، الهاتف، الاسم، العنوان، الهوية الوطنية |
| `popia` | قانون حماية المعلومات الشخصية | جنوب إفريقيا | البريد الإلكتروني، الهاتف، الاسم، العنوان، رقم الهوية |
| `privacy-act` | قانون الخصوصية 1988 | أستراليا | البريد الإلكتروني، الهاتف، الاسم، العنوان، Medicare، TFN |
| `pipeda` | قانون حماية المعلومات الشخصية والوثائق الإلكترونية | كندا | البريد الإلكتروني، الهاتف، الاسم، العنوان، SIN |

## التكوين لكل Preset

يحدد كل preset ما يلي:

- **أنواع الكيانات الإلزامية**: قواعد الكشف التي لا يمكن تعطيلها أثناء نشاط الـ preset.
- **عتبة الثقة**: الحد الأدنى لدرجة الثقة للكشف القائم على NER (افتراضي لكل preset).
- **الإعداد الإقليمي (Locale)**: الإعداد الإقليمي المرتبط بالتعرفات المحددة للغة.

## تكوين Presets

قم بتعيين الـ preset عبر ترويسة `X-AnonReq-Compliance-Preset` في الطلبات:

```bash
curl -H "X-AnonReq-Compliance-Preset: gdpr"   -H "Authorization: Bearer $ANONREQ_API_KEY"   ...
```

استعلم عن presets المتاحة وتكويناتها:

```bash
curl http://localhost:8000/v1/compliance/presets
```

## Presets متعددة

عند تحديد خيارات متعددة (مفصولة بفاصلة)، يكون التكوين الفعال عبارة عن اتحاد لجميع أنواع الكيانات المفروضة مع أعلى عتبة ثقة بين الخيارات المحددة.

## التحقق عند بدء التشغيل

إذا تم تكوين preset للامتثال عند بدء التشغيل، فإن البوابة تتحقق من أن جميع أنواع الكيانات المفروضة بالـ preset تحتوي على تعرفات نشطة. يتم رفض التكوين الذي يعطل نوعًا مفروضًا عند بدء التشغيل.
