           - Verify tenant A's rules never affect tenant B's decision
         - `test_fail_secure_on_cache_failure(tenant_id, classification, region)`:
           - When UsageLimiter/SpendController cache is unavailable → BLOCK decision
           - Provider forward count remains 0
         - `test_deny_dominates_allow(conflicting_rules, tenant_id)`:
           - When policies conflict, BLOCK/deny actions dominate ALLOW actions
           - Generate rule sets with conflicting priorities, verify BLOCK wins
         - `test_no_raw_pii_in_audit(generated_request_data, tenant_id)`:
           - For generated tenant IDs, session IDs, actor IDs, and policy inputs
           - Decisions remain tenant-scoped
           - Audit events never include raw generated sensitive strings in fields
         - `test_decision_deterministic(seed, tenant_id, rules)`:
           - Same input + same policy version → same decision every time
         - `test_rate_limit_counters_monotonic(tenant_id, increments)`:
           - Rate limit counters are monotonic within window (never decrease except reset)
