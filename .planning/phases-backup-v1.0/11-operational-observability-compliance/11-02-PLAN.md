<objective>
Implement SLO tracking and breach alerting infrastructure. This plan delivers: (1) SLO counter storage in Valkey with real-time reads, (2) the SLO computation engine for both fixed calendar windows (daily/monthly) and rolling windows (24h/30d), (3) the GET /v1/governance/status endpoint with per-SLO compliance, and (4) breach detection with configurable webhook delivery and dead letter queue.

Purpose: SREs need real-time visibility into SLO compliance and immediate alerting when thresholds are breached. The dual-window approach satisfies both compliance auditors (calendar-based) and operations teams (rolling).

Output: GET /v1/governance/status returns per-SLO compliance percentages and last breach timestamps. SLO breaches trigger webhook POST with 3x retry, failed deliveries queued to DLQ.
</objective>

       ```python
       @dataclass
       class SLOCounter:
           slo_name: str
           window_type: str    # "daily" | "monthly" | "24h" | "30d"
           window_key: str      # e.g., "2026-06-20" for daily
           numerator: int
           denominator: int

       @dataclass
       class SLOCompliance:
           slo_name: str
           target: float
           current: float       # percentage (0-100)
           compliant: bool
           window_type: str
           window_key: str
           last_breach: datetime | None

       class SLOEngine:
           def __init__(self, cache_manager: CacheManager, config: SLOConfig):
               self._cache = cache_manager
               self._config = config

           async def record_success(self, tenant_id: str, slo_name: str, latency_ms: int = 0) -> None:
               """Increment success counter for all configured windows."""

           async def record_failure(self, tenant_id: str, slo_name: str) -> None:
               """Increment failure counter for all configured windows."""

           async def compute_compliance(self, tenant_id: str, slo_name: str | None = None) -> list[SLOCompliance]:
               """Compute SLO compliance for all windows.
               For each window: numerator / denominator * 100, compare to target.
               Returns list of SLOCompliance objects."""

           async def get_all_compliance(self, tenant_id: str) -> dict[str, list[SLOCompliance]]:
               """Get compliance for all configured SLOs keyed by slo_name."""

           async def record_latency(self, tenant_id: str, latency_ms: int) -> None:
               """Record latency observation for P95 computation."""
       ```
       - Counters stored in Valkey using sorted sets (ZINCRBY) for time-based windows
       - Window key format: `slo:{tenant_id}:{slo_name}:{window_type}:{window_key}`
       - P95 computed from histogram stored in Valkey sorted set
       - For rolling windows, ZREMRANGEBYSCORE removes expired entries
       - Use `time.time()` as score for rolling window entries

       ```python
       @dataclass
       class BreachEvent:
           id: str
           slo_name: str
           target: float
           current_value: float
           window_type: str
           window_key: str
           tenant_id: str
           detected_at: datetime
           acknowledged: bool = False

       class BreachDetector:
           def __init__(self, slo_engine: SLOEngine, audit_chain: AuditChainService,
                        http_client: httpx.AsyncClient, config: WebhookConfig):
               self._slo_engine = slo_engine
               self._audit_chain = audit_chain
               self._http_client = http_client
               self._config = config

           async def evaluate(self, tenant_id: str) -> list[BreachEvent]:
               """Evaluate all SLOs for breaches. Returns list of new breaches.
               1. Get all compliance data from SLOEngine
               2. For each non-compliant SLO:
                  a. Check cooldown: was same SLO breached in last cooldown_seconds?
                  b. If cooldown active: skip
                  c. If new breach: create BreachEvent, fire webhook, log audit event
                  d. Store breach timestamp for cooldown tracking"""

           async def _fire_webhook(self, event: BreachEvent) -> bool:
               """POST breach event to configured webhook URL.
               Retry 3x with exponential backoff.
               On final failure: add to DLQ."""

           async def _deliver_to_dlq(self, event: BreachEvent) -> None:
               """Store failed delivery in Valkey DLQ list (LPUSH).
               DLQ key: breach_dlq:{tenant_id}
               Max entries: configurable (default 1000), trim on overflow."""

           async def get_dlq_entries(self, tenant_id: str, limit: int = 100) -> list[BreachEvent]:
               """Return DLQ entries (LRANGE)."""

           async def ack_dlq_entry(self, tenant_id: str, event_id: str) -> None:
               """Remove individual DLQ entry (LREM)."""

           async def retry_dlq(self, tenant_id: str) -> int:
               """Retry all DLQ entries. Returns count of successfully delivered."""
       ```
       - Webhook payload: `{"event_type": "slo_breach_detected", "slo_name": "...", "target": 99.9, "current": 98.5, "window": "...", "tenant_id": "...", "detected_at": "..."}`
       - Exponential backoff: `base * 2^attempt` seconds between retries
       - DLQ stored in Valkey as a list, trimmed to max_entries on each LPUSH
       - Audit event emitted via AuditChainService for every breach
       - BreachDetector called periodically by background task (every 60s)

|-----------|----------|-----------|-------------|-----------------|

| T-11-02-03 | Denial of Service | Webhook endpoint failure | mitigate | 3x retry with exponential backoff + DLQ prevents sender blocking. |
