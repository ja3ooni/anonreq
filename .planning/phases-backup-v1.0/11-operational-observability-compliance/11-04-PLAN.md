    Create the Prometheus and Grafana configuration:

    1. **docker/prometheus/prometheus.yml**:
       ```yaml
       global:
         scrape_interval: 15s
         evaluation_interval: 15s

       rule_files:
         - "rules/slo_alerts.yml"

       scrape_configs:
         - job_name: "anonreq"
           static_configs:
             - targets: ["anonreq:8080"]
           metrics_path: "/metrics"
           scheme: http

         - job_name: "postgres-exporter"
           static_configs:
             - targets: ["postgres-exporter:9187"]

         - job_name: "valkey-exporter"
           static_configs:
             - targets: ["valkey-exporter:9121"]
       ```

    2. **docker/prometheus/rules/slo_alerts.yml**:
       ```yaml
       groups:
         - name: slo_alerts
           rules:
             - alert: SLOSuccessRateBreach
               expr: anonreq_slo_success_rate < 99.9
               for: 5m
               labels:
                 severity: critical
               annotations:
                 summary: "SLO success rate below target (current: {{ $value }}%)"

             - alert: SLOP95LatencyBreach
               expr: anonreq_slo_p95_latency_ms > 100
               for: 5m
               labels:
                 severity: warning
               annotations:
                 summary: "P95 latency exceeds 100ms (current: {{ $value }}ms)"

             - alert: FailSecureRateBreach
               expr: anonreq_slo_fail_secure_rate > 0.1
               for: 5m
               labels:
                 severity: critical
               annotations:
                 summary: "Fail-secure rate exceeds 0.1% (current: {{ $value }}%)"
       ```

    3. **docker/grafana/datasources/prometheus.yml**:
       ```yaml
       apiVersion: 1
       datasources:
         - name: Prometheus
           type: prometheus
           access: proxy
           url: http://prometheus:9090
           isDefault: true
       ```

    4. **docker/grafana/dashboards/slo_dashboard.json** — Grafana dashboard JSON with:
       - Row 1: SLO compliance gauges (4 panels: success rate, P95 latency, fail-secure, audit write)
       - Row 2: SLO compliance time series (4 panels showing trends over past 7/30 days)
       - Row 3: Breach events table (last 50 breach events with timestamp, SLO, value)
       - Row 4: Gateway metrics (RPM, latency histogram, active sessions)
       - Each panel: title, Prometheus query, visualization type, thresholds

    5. **docker/grafana/dashboards/audit_dashboard.json** — Audit dashboard:
       - Row 1: Audit events by event_type (bar chart, last 30 days)
       - Row 2: Config change timeline (operator_id, change_type)
       - Row 3: Chain anchor status (last anchor timestamp, event count, verification)
       - Row 4: Export history (monthly export sizes, status)

    Use Grafana Dashboard JSON model (can be exported from Grafana UI or written manually).
  </action>

    Update Docker Compose with Phase 11 services:

    ```yaml
    services:
      # Existing services
      anonreq:
        # ... existing config ...
        environment:
          - ANONREQ_DATABASE_URL=postgresql+asyncpg://anonreq:anonreq@postgres:5432/anonreq
          - ANONREQ_MINIO_ACCESS_KEY=${ANONREQ_MINIO_ACCESS_KEY:-minioadmin}
          - ANONREQ_MINIO_SECRET_KEY=${ANONREQ_MINIO_SECRET_KEY:-minioadmin}
          - ANONREQ_BREACH_WEBHOOK_URL=${ANONREQ_BREACH_WEBHOOK_URL:-}
          - ANONREQ_ANCHOR_SIGNING_KEY=${ANONREQ_ANCHOR_SIGNING_KEY:-}
        depends_on:
          postgres:
            condition: service_healthy
          minio:
            condition: service_healthy
        volumes:
          - ./alembic:/app/alembic  # For runtime migrations

      postgres:
        image: postgres:16-alpine
        environment:
          POSTGRES_USER: anonreq
          POSTGRES_PASSWORD: anonreq
          POSTGRES_DB: anonreq
        volumes:
          - postgres_data:/var/lib/postgresql/data
        healthcheck:
          test: ["CMD-SHELL", "pg_isready -U anonreq"]
          interval: 5s
          timeout: 5s
          retries: 5
        ports:
          - "5432:5432"
        profiles: ["observability"]

      postgres-exporter:
        image: prometheuscommunity/postgres-exporter:v0.15.0
        environment:
          DATA_SOURCE_NAME: "postgresql://anonreq:anonreq@postgres:5432/anonreq?sslmode=disable"
        depends_on:
          postgres:
            condition: service_healthy
        profiles: ["observability"]

      minio:
        image: minio/minio:latest
        command: server /data --console-address ":9001"
        environment:
          MINIO_ROOT_USER: minioadmin
          MINIO_ROOT_PASSWORD: minioadmin
        volumes:
          - minio_data:/data
        healthcheck:
          test: ["CMD", "curl", "-f", "http://localhost:9000/minio/health/live"]
          interval: 10s
          timeout: 5s
          retries: 5
        ports:
          - "9000:9000"
          - "9001:9001"
        profiles: ["observability"]

      prometheus:
        image: prom/prometheus:v2.53.0
        volumes:
          - ./docker/prometheus:/etc/prometheus
          - prometheus_data:/prometheus
        command:
          - "--config.file=/etc/prometheus/prometheus.yml"
          - "--storage.tsdb.retention.time=30d"
        ports:
          - "9090:9090"
        depends_on:
          anonreq:
            condition: service_started
        profiles: ["observability"]

      grafana:
        image: grafana/grafana:11.0.0
        environment:
          - GF_AUTH_ANONYMOUS_ENABLED=true
          - GF_AUTH_ANONYMOUS_ORG_ROLE=Viewer
          - GF_DASHBOARDS_DEFAULT_HOME_DASHBOARD_PATH=/etc/grafana/dashboards/slo_dashboard.json
        volumes:
          - ./docker/grafana/datasources:/etc/grafana/provisioning/datasources
          - ./docker/grafana/dashboards:/etc/grafana/provisioning/dashboards
          - grafana_data:/var/lib/grafana
        ports:
          - "3000:3000"
        depends_on:
          - prometheus
        profiles: ["observability"]

      valkey-exporter:
        image: oliver006/redis_exporter:v1.61.0
        environment:
          REDIS_ADDR: redis://valkey:6379
        depends_on:
          valkey:
            condition: service_started
        profiles: ["observability"]

    volumes:
      postgres_data:
      minio_data:
      prometheus_data:
      grafana_data:
    ```
    - New services use `profiles: ["observability"]` so they're optional for basic deployments
    - `docker compose --profile observability up` starts the full stack
    - Health checks on postgres and minio ensure correct startup ordering
    - Persistent volumes for postgres, minio, prometheus, grafana

       - Fixed: daily (calendar), monthly (calendar)
       - Rolling: 24h, 30d

       ## Breach Response
       | SLO | Severity | Response |
       |-----|----------|----------|
       | Success Rate < 99.9% | Critical | Check upstream providers, Valkey connectivity |
       | P95 > 100ms | Warning | Check resource utilization, connection pools |
       | Fail-secure > 0.1% | Critical | Investigate detection pipeline, auth errors |
       | Audit write < 99.99% | Critical | Check PostgreSQL connectivity, disk space |

       ## Verification
       - curl http://localhost:8080/v1/governance/status
       - Check webhook delivery logs at /v1/governance/breaches
       - Review DLQ for failed webhook deliveries

       ## Classification
       | Severity | Definition | Response Time |
       |----------|------------|---------------|

       | S2 | Service degradation | ≤ 4 hours |
       | S3 | Security anomaly | ≤ 24 hours |

       1. Detection (automated alert, user report, audit review)
       2. Triage (confirm, classify, assign)
