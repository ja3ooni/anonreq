<objective>
Implement the TextExtractor, Classification Engine, and hybrid Detection Engine for the AnonReq pipeline. This plan delivers: (1) TextExtractor — recursive JSON walker extracting TextNodes from OpenAI chat payloads (all roles), (2) ClassificationEngine — YAML-based 4-tier rule evaluation (BLOCK/ROUTE_LOCAL/ANONYMIZE/PASS) with action-based precedence, (3) RegexDetector — pre-compiled patterns for structured PII with Luhn validation for credit cards, (4) PresidioClient — async HTTP client to the Presidio Analyzer sidecar with per-TextNode concurrency via asyncio.gather and semaphore, (5) SpanArbiter — regex+NIR merge with specificity-ranked overlap resolution, and (6) ExclusionList — exact-match and wildcard suppression.

Purpose: Classification must run before detection per ROADMAP hardening decision. The detection hybrid (regex + NER with arbitration) is the core PII identification layer. Without it, tokenization has nothing to replace.

Output: Working classification and detection modules where `ClassificationEngine.classify(text_nodes)` returns action and matched rules, and `SpanArbiter.merge(regex_results, ner_results)` produces the definitive detection list.
</objective>

       - `class SpanArbiter`:
         - `@staticmethod merge(regex_results: list[dict], ner_results: list[dict]) -> list[dict]`:
           - Tag each result with `_source` field: "regex" or "ner"
           - Combine all results into one list
           - Sort by start position ascending, then score descending
           - Iterate through combined results, building merged list:
             - For each span, check overlap with already-accepted spans
             - Use `_overlap_type()` helper to classify overlap
             - Exact overlap → regex wins (replace accepted if new span is regex)
             - Nested → most specific entity type wins (higher ENTITY_SPECIFICITY value)
             - Partial → most specific entity type wins
             - Non-overlapping → append to merged list
           - Strip `_source` tag before returning
           - Return filtered list sorted by start position ascending
         - `@staticmethod _overlap_type(a: dict, b: dict) -> str | None`:
           - Calculate start/end overlap between two spans
           - Return "exact" (same start and end), "nested" (one within other), "partial" (partial overlap), or None (no overlap)
         - The merge logic handles the case where regex found the same entity as NER but with different span boundaries — regex wins per D-40
