       ```yaml
       mnpi:
         ticker_pattern: '\b[A-Z]{1,4}(\.\w{1,2})?\b'
         ticker_exchanges: ["NYSE", "NASDAQ", "AMEX"]
         deal_codename_patterns:
           - '\b(Project|Operation|Initiative)\s+[A-Z][a-zA-Z]{2,}\b'
       ```

    2. **src/anonreq/detection/recognizers/mnpi.py**:
       - `MNPIConfig` dataclass loaded from YAML: ticker_pattern, deal_codename_patterns, excluded_words, min_ticker_length, score
       - `class MNPIRecognizer(EntityRecognizer)`:
         - Extends Presidio's `EntityRecognizer` (pattern-based)
         - `__init__(self, config: MNPIConfig, restricted_names: RestrictedNamesManager | None = None)`: stores config and optional restricted names
         - `def load(self) -> list[RecognizerResult]`: loads patterns into Presidio
         - `def analyze(self, text: str, entities: list[str], nlp_artifacts) -> list[RecognizerResult]`:
           - Runs ticker regex → yields `MNPI_TICKER` results
           - Runs deal codename regex → yields `MNPI_DEAL` results
           - If restricted_names available, checks text for restricted name matches → yields `MNPI_RESTRICTED_NAME` results
           - Filters out excluded_words
           - Deduplicates overlapping matches (shorter match loses)
       - `def create_mnpi_bundle(config_path: str, restricted_names_mgr: RestrictedNamesManager | None = None) -> list[EntityRecognizer]`:
         - Loads config from YAML, creates MNPIRecognizer instance
         - Returns list for registration in the Presidio analyzer pipeline

    3. **Add to src/anonreq/models/detection.py**:
       - Add `MNPI_TICKER`, `MNPI_DEAL`, `MNPI_RESTRICTED_NAME` to entity types
       - Add `MNPI_POLICY_ACTION` type: `Literal["anonymize", "flag", "block", "quarantine"]`

    4. **Register in detection pipeline** (`src/anonreq/detection/pipeline.py`):
       - Load MNPI recognizer bundle at startup alongside core recognizers
       - MNPI runs after core Presidio pipeline (as additional recognizer)

           - Returns object path

           - Downloads and returns event JSON

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Detection pipeline → MNPI recognizer | PII/MNPI text analyzed in-memory |
| Gateway → MinIO | MNPI audit events uploaded to WORM bucket |
| Restricted names file → Filesystem | YAML config read at startup and hot-reload |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01-01 | Information Disclosure | MNPI recognizer | mitigate | All detection happens in-memory; only hashed values stored in audit |
| T-15-01-02 | Tampering | MinIO WORM bucket | mitigate | Object lock with COMPLIANCE mode prevents delete/overwrite per SEC 17a-4 |
| T-15-01-03 | Tampering | Restricted names YAML | mitigate | yaml.safe_load() prevents code injection; hot-reload uses file mtime check |
| T-15-01-04 | Spoofing | Ticker symbol false positives | accept | Excluded_words list reduces false positives; configurable score threshold |

</threat_model>
