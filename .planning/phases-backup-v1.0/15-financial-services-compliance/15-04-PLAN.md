           - DORA: incidents, provider inventory, SLO status

           - `evidence`: references to relevant records/events
           - `summary`: overall compliance status (compliant / partial / non_compliant)

         - Generate random uppercase words of length 1-4

         - Verify MnpiAuditEvent.detected_value_hash is never the original value

    2. **tests/property/test_financial_crime_invariants.py**:
       - **Boost bounded invariant**: For any entity confidence score and any text, the boosted score never exceeds 1.0 and never goes below 0.0
         - Generate random scores [0.0, 1.0] and random positions
         - Apply boost → verify result in [0.0, 1.0]
       - **Boost only-financial invariant**: Non-financial entity types never have their score modified by context booster
         - Generate random non-financial entity types (EMAIL, PHONE, etc.)
         - Apply boost → verify score unchanged
       - **Proximity correctness invariant**: Entities more than 50 chars from any high-risk word never receive a boost
         - Generate random entity positions far from word positions
         - Apply boost → verify score unchanged
       - **AML webhook threshold invariant**: AML webhook fires iff confidence >= threshold
         - Generate random confidences and thresholds
         - Verify: fire = (confidence >= threshold)
