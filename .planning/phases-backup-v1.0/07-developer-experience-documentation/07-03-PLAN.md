    - "examples/**"

    Jobs:

    1. markdown-lint:
    - runs-on: ubuntu-latest

      (config: inline .markdownlint.json with default rules, MD013 line-length disabled,
      MD033 no-inline-html disabled)

    2. link-check:
    - runs-on: ubuntu-latest
    - steps: checkout, npm install -g lychee (version 0.2.12),
      run: lychee --no-progress --exclude-mailto "docs/**/*.md" "README.md"

    3. mermaid-validate:
    - runs-on: ubuntu-latest
    - steps: checkout, npm install -g @mermaid-js/mermaid-cli (version 11.15.0),
      run: mmdc -i docs/architecture.mmd -o docs/architecture.svg,
      run: mmdc -i docs/architecture.mmd -o docs/architecture.png -s 2
    - Per D-204: CI validates diagram generation

    4. changelog-validate:
    - runs-on: ubuntu-latest
    - steps: checkout, pip install python-kacl,
      run: kacl-cli verify --json CHANGELOG.md
    - Add conditional step for version bump check on code changes (not docs-only):
      if: github.event.head_commit.message does not start with 'docs:'
      run: compare kacl-cli version output with latest git tag

    5. openapi-sync:
    - runs-on: ubuntu-latest
    - steps: checkout, setup-python v5 with 3.12, pip install -e .,
      run: python scripts/export_openapi.py > docs/openapi.json,
      npm install -g @redocly/cli (version 2.34.0),
      run: redocly lint docs/openapi.json,
      run: git diff --exit-code docs/openapi.json (fail if OpenAPI spec out of sync)
    - Per D-210: OpenAPI auto-generated from FastAPI in CI

    6. quickstart-execute:
    - runs-on: ubuntu-latest
    - services: docker
    - steps: checkout, start gateway via 01-start-gateway.sh,
      run: 02-basic-anonymization.sh,
      run: 03-cleanup.sh
    - Per D-195: quickstart scripts executed on every PR with verification assertions

    7. translation-drift:
    - runs-on: ubuntu-latest
    - steps: checkout, run: python scripts/check-translation-drift.py
    - Continue on error (warning, not blocking per D-207)

    - name: "Docs Nightly", on schedule (cron: 0 6 * * * — daily at 06:00 UTC).
    - Jobs:
      1. full-build: checkout, run all jobs from docs-ci.yml plus:

           available via secrets.DEEPL_API_KEY)
         - Cross-platform check: test markdownlint on ubuntu + macos-latest
         - Generate all diagram variants: SVG, PNG, PDF

    - Shebang `#!/usr/bin/env python3`
    - Traverse docs/{en}/ for .md files, compare git log -1 --format=%ct timestamps
      with corresponding docs/{de}/ files.
    - Report WARNING for staled translations or missing files.
    - Exit 0 always (non-blocking warning per D-207).
    - Support TARGET_LANGS = ["de"] initially, extendable list.
    - Per D-191: English source, generated translations, drift detectable via timestamps.

    Create scripts/translate-docs.py:
    - Shebang `#!/usr/bin/env python3`
    - Read English markdown files from docs/en/.
    - Call DeepL API (free tier) to translate content to German.
    - Write translated content to docs/de/ preserving directory structure.
    - Use DEEPL_API_KEY from environment variable.
    - If API key not set, print "DEEPL_API_KEY not set — skipping translation"
      and exit 0 (non-blocking).
    - Preserve markdown formatting, code blocks, and inline links during translation.
      Use DeepL's tag handling to protect inline code spans and links.
    - Per D-191: DE docs are generated artifacts, never manually edited.
    - Use requests library for DeepL API v2: POST /v2/translate with target_lang DE.
    - Fall back to English-only if API key unavailable per RESEARCH.md Open Questions.

    Per D-206: required CI checks (markdown lint, link validation, Mermaid validation,
    diagram generation, example execution, quickstart execution, OpenAPI sync).
    Per D-207: non-blocking warnings (translation drift, CHANGELOG reminders).
    Per D-208: nightly CI with full build across all languages.
    Per D-209: CHANGELOG format validation + version bump check.
