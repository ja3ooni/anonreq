  <files>
    examples/quickstart/01-start-gateway.sh

    examples/quickstart/03-cleanup.sh
    examples/datasets/sample-pii.json
    examples/datasets/expected-results.json
  </files>
  <action>
    Create three executable bash scripts under examples/quickstart/ using the trap-based
    pass/fail pattern from RESEARCH.md Pattern 1. Each script uses `set -euo pipefail`
    and a trap that sets PASSED=false on ERR. Scripts reference the repository root via
    PROJECT_ROOT derived from `$(dirname "$0")/../..`. Use ANONREQ_API_KEY env var
    defaulting to test-key-0123456789abcdef. Each script prints === Phase: descriptions
    for each step, measures elapsed time, and exits 0 on pass or 1 on fail.

    Script 01-start-gateway.sh:
    - Runs `docker compose up -d --wait --wait-timeout 60` from PROJECT_ROOT
    - Verifies health via `curl -s -o /dev/null -w "%{http_code}" http://localhost:8000/health`
      asserts 200. On failure, dumps `docker compose logs anonreq --tail=20`.
    - Prints PASS message with gateway URL.

    Script 02-basic-anonymization.sh:
    - Depends on gateway already running (01 must precede).
    - Sends POST to /v1/chat/completions with sample PII message containing
      email "john.doe@example.com" and phone "+1-555-123-4567".
    - Uses response piping to python3 -c snippet that imports json/sys, parses response,
      asserts `[EMAIL_1]` in content, asserts `[PHONE_1]` in content.
    - Prints the tokenized content on success.

    Script 03-cleanup.sh:
    - Runs `docker compose down` from PROJECT_ROOT.
    - Checks `docker compose logs anonreq 2>&1 | grep -c -i error` — asserts zero.
    - Prints PASS or FAIL based on error count.

    Dataset files under examples/datasets/:
    sample-pii.json: JSON array of objects with fields: `field` (description), `value`
    (PII string), `type` (entity type), `locale` (language code). Include at least:
    email, phone, credit card, IBAN, IP address, URL, date of birth, name, org, address,
    and a German Steuer-ID.
    expected-results.json: JSON object keyed by the sample-pii.json `field` values,
    with `tokenized` field showing the expected `[TYPE_N]` output for each, and
    `entity_type` for validation.

    Mark all scripts executable: chmod +x.
    Per D-194: quickstarts are executable scripts, not markdown with inline commands.
    Per D-195: scripts must assert gateway starts, health 200, anonymization succeeds,
    restoration succeeds, zero errors in logs.
    Per D-201: shared dataset directory for all SDK examples.
  </action>
  <verify>
    <automated>shellcheck examples/quickstart/*.sh && python3 -m json.tool examples/datasets/sample-pii.json && python3 -m json.tool examples/datasets/expected-results.json</automated>
  </verify>
  <done>
    3 quickstart scripts pass shellcheck, 2 dataset files are valid JSON, scripts follow trap-based pass/fail pattern
  </done>
