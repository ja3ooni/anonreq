<objective>
Implement the health monitoring and configuration management layer that ties all sink implementations together. This wave delivers: (1) periodic per-sink health probes with configurable interval, (2) GET /v1/admin/soc/integration/status endpoint with RBAC, (3) config/soc-sinks.yaml schema with env var and file-based secret references, and (4) Prometheus gauges for per-sink health visibility.

Purpose: Without health monitoring, operators cannot detect SIEM integration failures. Without config management, sink auth credentials would leak into version control. This wave makes the entire SOC integration operational.

Output: Operators can inspect integration health at GET /v1/admin/soc/integration/status and configure all sinks via config/soc-sinks.yaml with secure secret references.
</objective>

       - `class SinkHealthMonitor`:
         - `__init__(self, router: SinkRouter, interval: int = 60, metrics_registry=None)`
         - `_tasks: dict[str, asyncio.Task]` — per-sink probe tasks
         - `_status_cache: dict[str, SinkStatus]` — latest status per sink
         - `async def start(self)`: start per-sink probe tasks
         - `async def stop(self)`: cancel all probe tasks
         - `async def _probe_loop(self, sink_name: str)`:
           - Loop every `interval` seconds
           - Call `sink.health_check()`
           - Update `_status_cache` with result
           - On status change (reachable→unreachable or vice versa): emit audit event soc_sink_reachable / soc_sink_unreachable
           - Update Prometheus gauge `anonreq_soc_sink_healthy` (1=healthy, 0=unhealthy)
         - `def get_status(self) -> dict[str, SinkStatus]`: return cached statuses
         - `def get_aggregate_status(self) -> str`: "healthy" if all enabled sinks reachable, "degraded" if any unreachable, "unknown" if no sinks
