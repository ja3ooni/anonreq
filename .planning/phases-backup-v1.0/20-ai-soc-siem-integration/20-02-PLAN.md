<objective>
Implement the Splunk HEC and QRadar CEF SIEM sinks — the first two output formatters in the SOC integration layer. Splunk HEC delivers JSON envelopes via HTTP POST with Splunk auth token. QRadar CEF formats events as Common Event Format strings over syslog TCP/UDP. Both sinks implement the shared SinkBase interface for uniform lifecycle management through the SinkRouter.

Purpose: These are the two most commonly deployed SIEM sinks in enterprise SOC environments. Wave 2 establishes the sink abstraction pattern reused by Waves 3 and 4.

Output: `SplunkHECSink` and `QRadarCEFSink` with format/send/health_check methods, unit tests passing with respx HTTP mocks and asyncio syslog mocks.
</objective>

       ```python
       from dataclasses import dataclass
       from typing import Protocol, runtime_checkable
       from datetime import datetime

       @dataclass
       class SinkStatus:
           healthy: bool
           reachable: bool
           last_successful_delivery: datetime | None = None
           last_error: str | None = None
           buffer_size: int = 0

       @runtime_checkable
       class SinkBase(Protocol):
           name: str
           sink_type: str
           enabled: bool

           async def start(self) -> None: ...
           async def stop(self) -> None: ...
           async def send_event(self, event: 'NormalizedEvent') -> bool: ...
           async def health_check(self) -> SinkStatus: ...
           async def format_event(self, event: 'NormalizedEvent') -> str | bytes: ...
       ```

    2. **src/anonreq/soc/router.py**:
       - `class SinkRouter`:
         - `__init__(self)`: maintains `_sinks: dict[str, SinkBase]`
         - `register(self, sink: SinkBase)`: add a sink
         - `async def fan_out(self, event: NormalizedEvent) -> None`: iterate all enabled sinks, call send_event
         - `def get_sinks(self) -> dict[str, SinkStatus]`: return current status of all sinks
         - `async def start_all(self)`: call start() on all registered sinks
         - `async def stop_all(self)`: call stop() on all registered sinks
  </action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SplunkHECSink → Splunk HEC endpoint | Normalized events transmitted over HTTPS to external SIEM |
| QRadarCEFSink → QRadar syslog | CEF messages transmitted over TCP/UDP syslog |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-02-01 | Information Disclosure | Splunk HEC HTTPS transport | mitigate | TLS verification enabled by default. Token in Authorization header (not URL). |
| T-20-02-02 | Information Disclosure | QRadar syslog transport | accept | Syslog lacks native encryption. Deployment responsible for network-level encryption (IPsec/TLS tunnel). |
| T-20-02-03 | Spoofing | Splunk HEC auth token | mitigate | Token resolved from env/file secrets, never in logs or config YAML. |
| T-20-02-04 | Tampering | CEF message escaping | mitigate | Special characters (|, =, \, newline) escaped in extension field values. |
</threat_model>
