<objective>
Implement the generic webhook SIEM sink and the per-sink buffer/retry infrastructure. The webhook sink provides a flexible integration point for custom SIEM platforms not covered by the five fixed sinks. The buffer/retry system ensures event delivery resilience with in-memory queuing, LRU eviction, and exponential backoff — all without ever blocking request processing.

Purpose: Every sink needs buffer/retry (D-017 through D-021). Centralizing this in a reusable SinkBuffer avoids per-sink duplication. The webhook sink future-proofs against SIEM vendor lock-in.

Output: WebhookSink with Jinja2-subset templating and SinkBuffer with LRU eviction + exponential backoff, all tests passing.
</objective>

    Implement the per-sink buffer and retry manager per D-017 through D-021, 20-ARCHITECTURE.md.

    1. **src/anonreq/soc/buffer.py**:
       - `BackoffConfig` dataclass: initial=1, multiplier=2, max=60, jitter=0.1, max_retries=5
       - `class SinkBuffer`:
         - `__init__(self, sink: SinkBase, maxsize: int = 10000, backoff: BackoffConfig | None = None, audit_logger=None, metrics_registry=None)`
         - `_queue: asyncio.Queue[tuple[int, NormalizedEvent]]` — queue with monotonic counter for LRU
         - `_counter: int` — monotonic counter for insertion order
         - `_overflow_emitted: bool` — prevent duplicate overflow audit events
         - `async def put(self, event: NormalizedEvent)`:
           - If queue full: drop oldest via LRU (count oldest insertion index, drain oldest), increment overflow counter
           - Use `put_nowait` for non-blocking
         - `async def _retry_loop(self)`:
           - Background asyncio task
           - Get event from queue, attempt send_event with backoff
           - On success: log soc_event_forwarded, continue
           - On failure: compute backoff, sleep, retry
           - On max retries exhausted: log soc_event_dropped, drop event
         - `async def start(self)`: start _retry_loop task
         - `async def stop(self)`: cancel task, drain remaining events
         - `def current_size(self) -> int`: queue size
         - Prometheus: `anonreq_soc_buffer_size` gauge (per sink label), `anonreq_soc_buffer_overflow_total` counter

       Backoff formula per ARCHITECTURE.md:
       ```python
       import random
       def _backoff_delay(self, attempt: int) -> float:
           base = min(self._backoff.initial * (self._backoff.multiplier ** attempt), self._backoff.max)
           jitter = base * self._backoff.jitter
           return base + random.uniform(-jitter, jitter)
       ```
