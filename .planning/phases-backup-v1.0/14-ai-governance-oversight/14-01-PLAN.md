    Create the governance record layer per decisions D-001 through D-005.

    1. **src/anonreq/models/governance.py** — Pydantic models:
       - `GovernanceOfficerRole` enum: `GOVERNANCE`, `RISK`, `COMPLIANCE`, `SECURITY`
       - `GovernanceOfficer(BaseModel)`: role (GovernanceOfficerRole), name (str), email (str)
       - `ReviewCycle(BaseModel)`: id (int), tenant_id (str), interval_days (int=90), last_review_date (datetime | None), next_review_date (datetime | None), status (str="active")
       - `GovernanceRecord(BaseModel)`: id (int), tenant_id (str), officers (list[GovernanceOfficer]), review_cycle (ReviewCycle), created_at (datetime), updated_at (datetime), status (str="active")
       - `RiskDimensionScore(BaseModel)`: dimension (str), severity (int=1-5), likelihood (int=1-5), overall_score (float), treatment_plan (str | None)
       - `RiskAssessment(BaseModel)`: id (int), tenant_id (str), governance_record_id (int), dimensions (list[RiskDimensionScore]), overall_risk_score (float), reassessment_required (bool=False), created_at (datetime), updated_at (datetime)
       - Add `model_config = {"extra": "ignore"}` to all models.

    2. **src/anonreq/governance/__init__.py** — import and re-export router and functions.

    3. **src/anonreq/governance/records.py** — async PostgreSQL CRUD:
       - `async def create_governance_record(db: AsyncSession, tenant_id: str, officers: list[GovernanceOfficer], interval_days: int = 90) -> GovernanceRecord`
         - Creates a ReviewCycle with last_review_date=None, next_review_date=utcnow()+interval_days
         - Creates GovernanceRecord linked to the review cycle
         - Returns the created record with IDs
       - `async def get_governance_record(db: AsyncSession, tenant_id: str) -> GovernanceRecord | None`
         - SELECT by tenant_id, joins review_cycle
       - `async def update_governance_record(db: AsyncSession, tenant_id: str, officers: list[GovernanceOfficer]) -> GovernanceRecord`
         - Updates officers, sets updated_at
       - `async def list_governance_records(db: AsyncSession, skip: int = 0, limit: int = 50) -> list[GovernanceRecord]`
       - `async def get_overdue_reviews(db: AsyncSession) -> list[GovernanceRecord]`
         - SELECT where review_cycle.next_review_date < now() AND status = 'active'
       - `async def get_upcoming_reviews(db: AsyncSession, days: int = 30) -> list[GovernanceRecord]`
         - SELECT where next_review_date BETWEEN now() AND now()+days
       - Use SQLAlchemy async ORM or raw asyncpg queries matching Phase 11 patterns.

    4. **src/anonreq/governance/router.py** — FastAPI admin endpoints:
       - `GET /v1/governance/records` — list all governance records (paginated)
         - Auth: admin role required
       - `GET /v1/governance/records/{tenant_id}` — get specific tenant record
       - `PUT /v1/governance/records/{tenant_id}` — update officers
       - `GET /v1/governance/status` — returns:
         ```json
         {
           "total_records": N,
           "overdue_reviews": N,
           "upcoming_reviews_30d": N,
           "overdue_by_tenant": [{"tenant_id": "...", "days_overdue": N}]
         }
         ```
       - `POST /v1/governance/records/{tenant_id}/reviews` — complete a review (updates dates)
       - All endpoints emit audit events per 14-SECURITY-ACCEPTANCE.md (governance_record_updated)
