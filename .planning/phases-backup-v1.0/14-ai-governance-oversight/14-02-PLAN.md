    - Test 5: rollback_version creates new version that restores target state

    Implement append-only governance object versioning per D-015 through D-018.

    1. **src/anonreq/governance/versioning.py**:
       - `GOVERNANCE_OBJECT_TYPES = ["policy", "provider", "preset"]`
       - `VersionRecord` dataclass: id, object_type, object_id, version_number, snapshot (JSON), diff_from_previous (JSON | None), approval_record (dict | None), operator (str), created_at (datetime)
       - `async def create_version(db: AsyncSession, object_type: str, object_id: str, snapshot: dict, operator: str, approval_record: dict | None = None) -> VersionRecord`:
         - Determines version_number: max existing + 1 (or 1 if first)
         - If previous version exists, computes diff via `compute_diff(previous.snapshot, snapshot)`
         - Stores as new row — never update existing
         - Returns created VersionRecord
       - `async def get_versions(db: AsyncSession, object_type: str, object_id: str) -> list[VersionRecord]`:
         - Returns all versions ordered by version_number asc
       - `async def get_version(db: AsyncSession, object_type: str, object_id: str, version_number: int) -> VersionRecord | None`
       - `async def diff_versions(db: AsyncSession, object_type: str, object_id: str, from_version: int, to_version: int) -> dict`:
         - Loads both versions, returns JSON Patch diff
       - `async def rollback_version(db: AsyncSession, object_type: str, object_id: str, target_version: int, operator: str, approval_record: dict | None = None) -> VersionRecord`:
         - Loads target version snapshot
         - Creates NEW version (not modifying existing) with that snapshot
         - Sets the object's current state to target snapshot
         - Emits `version_rollback` audit event
       - `def compute_diff(a: dict, b: dict) -> dict`:
         - Uses `jsonpatch.make_patch(a, b)` or manual recursive diff
         - Returns JSON Patch structure `[{"op": "replace", "path": "/field", "value": ...}]`

    2. **tests/test_governance_versioning.py**:
       - Test version creation sequence and numbering
       - Test diff computation between versions
       - Test rollback creates new version with restored state
       - Test append-only enforcement (no overwrite)
       - Test approval history stored with each version
