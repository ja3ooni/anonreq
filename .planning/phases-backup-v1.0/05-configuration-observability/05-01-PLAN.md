<objective>
Deliver operational observability for the AnonReq gateway with Prometheus metrics and P95 latency validation under load. This plan creates: (1) the Prometheus metrics module with 8 counters/histograms/gauges, (2) FastAPI middleware for request timing and label population, (3) instrumentation points in DetectionStage, ForwardingGuard, RestorationStage, fail-secure paths, and audit logger, (4) the GET /metrics endpoint returning Prometheus text format, and (5) a k6 load test script for validating P95 overhead ≤ 100ms at 50 concurrent users.

Purpose: Without metrics, operators cannot monitor gateway health, detect performance regressions, or validate the P95 latency target (PIPE-06). Without load testing, the P95 claim cannot be verified.

Output: A running gateway where `curl localhost:8080/metrics` returns all 8 metric families with live values and `k6 run tests/load/benchmark.js --vus 50 --duration 60s` produces a load test report.
</objective>

    1. **src/anonreq/monitoring/middleware.py**:
       ```python
       import time
       from fastapi import Request, Response
       from starlette.middleware.base import BaseHTTPMiddleware, RequestResponseEndpoint
       from . import metrics

       class MetricsMiddleware(BaseHTTPMiddleware):
           async def dispatch(self, request: Request, call_next: RequestResponseEndpoint) -> Response:
               # Record request receipt time on request.state for use by pipeline stages
               request.state.request_receipt_time = time.monotonic()
               request.state.provider_dispatch_time = None

               response = await call_next(request)

               # Determine labels from request/response state
               endpoint = request.url.path
               status_code = str(response.status_code)
               provider = getattr(request.state, "provider", "unknown")
               classification = getattr(request.state, "classification", "unknown")

               metrics.requests_total.labels(
                   endpoint=endpoint,
                   status_code=status_code,
                   provider=provider,
                   classification=classification
               ).inc()

               return response
       ```
       The middleware stores `request_receipt_time` on `request.state` for retrieval by pipeline stages.

                          'meeting', 'report', 'analysis', 'project', 'strategy'];
           let prompt = 'Please analyze the following: ';

           prompt += 'My email is user' + Math.floor(Math.random() * 1000) +
                     '@company.com and phone is +1-555-' +
                     String(Math.floor(Math.random() * 9000) + 1000) + '.';
           return prompt;

               model: 'gpt-4',
               messages: [{ role: 'user', content: generatePrompt(PROMPT_SIZE) }],
               stream: false,

                   'Content-Type': 'application/json',

               timeout: '30s',

               'status is 200 or 500': (r) => r.status === 200 || r.status === 500,
               'response time < 30s': (r) => r.timings.duration < 30000,
           });

           sleep(0.1);
       }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internal network → /metrics | Unauthenticated Prometheus scrape endpoint exposed on gateway port |
| Request metadata → metric labels | Provider name, classification action, endpoint path, and status code flow into label values |
| k6 load test → gateway | High-concurrency traffic generated by load test tool |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Information Disclosure | /metrics endpoint | mitigate | Documented as internal-network-only in deployment guide. No auth in MVP because Prometheus scrapers cannot easily handle bearer tokens. Future: mTLS or network policy. |
| T-05-01-02 | Information Disclosure | metric labels | mitigate | AG-15 enforced: no PII, entity values, request content, tenant_id, request_id, or session_id in any label. Label values drawn from controlled enums (endpoint, status_code, provider, classification, failure_type, entity_type, locale). |
| T-05-01-03 | Tampering | metric counter overflow | accept | Prometheus client uses float64 counters. For MVP scale (≤ 1M req/day), float64 precision sufficient. Monitor for counter reset at Phase 11. |
| T-05-01-04 | Denial of Service | load test hitting production | accept | Load test runs against isolated test instance. Documented in README.md. |
| T-05-01-SC | Tampering | prometheus_client package | mitigate | prometheus_client 0.25+ verified in Phase 1 package audit — [OK] verdict, 10yr history, 50M+ downloads, official Prometheus client. |
</threat_model>
