<objective>
Implement locale negotiation for the AnonReq gateway. This plan delivers: (1) LocaleNegotiator that parses the X-AnonReq-Locale header, resolves locale codes to LocaleBundle instances via LocaleRegistry, and handles fallback and error cases, (2) RecognizerMerger that creates a unified recognizer set from universal + locale-specific recognizers, deduplicating by entity type with highest confidence winning, (3) fallback logic for missing/malformed locales per LOCL-03, LOCL-04, and D-116 through D-118, (4) audit_metadata.locale field in ProcessingContext, (5) integration into the pipeline as a LocaleNegotiationStage before Detection, (6) updates to DetectionProvider to accept the merged recognizer set, and (7) unit + integration tests for all negotiation paths.

Purpose: Without locale negotiation, the gateway cannot activate locale-specific PII detection based on the client's locale header. All multi-locale support (LOCL-02 through LOCL-05) depends on this plan.

Output: A working `X-AnonReq-Locale: de-DE` header that triggers German Steuer-ID detection with checksum validation, and `X-AnonReq-Locale: de-DE, fr-FR` that merges German and French recognizers.
</objective>

         - was_fallback: bool = False

       - `class LocaleNegotiationError(Exception)`:
         - Constructor takes message, status_code=400, supported_locales list

       - `class LocaleNegotiator`:
         - `__init__(self, locale_registry: LocaleRegistry, universal_locale: str = "en")`
         - `def parse_header(self, header_value: str | None) -> HeaderParseResult`:
           - If None or empty: return HeaderParseResult([], was_fallback=True)
           - Split by comma, strip whitespace per entry
           - Filter empty entries (drop, log warning)
           - Cap at 10 entries (truncate, log warning)
           - Return HeaderParseResult with parsed codes
         - `def resolve_locales(self, parse_result: HeaderParseResult) -> list[LocaleBundle]`:
           - For each code in parse_result.locale_codes:
             - If LocaleRegistry.get(code) returns a bundle → use it
             - If not found → add to dropped list, log warning, continue
           - If result is empty (no valid locales found in single-locale request):
             - Raise LocaleNegotiationError with status 400 and supported list
           - If result is empty (no valid locales in multi-locale request):
             - Fall back to universal (en), set was_fallback=True, log warning
           - If single-locale request with unknown code:
             - Raise LocaleNegotiationError with status 400
         - `def negotiate(self, header_value: str | None) -> tuple[list[LocaleBundle], HeaderParseResult]`:
           - Combines parse_header + resolve_locales
           - Returns (bundles, parse_result) tuple

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering | Locale header injection | mitigate | AG-13: locale resolution is deterministic. Malformed locale → HTTP 400. Unknown in multi-locale → drop that entry, log, continue. No code execution from header. |
| T-04-02-02 | Information Disclosure | audit_metadata locale field | mitigate | D-136: Only effective locale codes logged (comma-separated). Raw header not logged. No sensitive info in locale codes. |
| T-04-02-03 | Tampering | RecognizerMerger confidence override | mitigate | AG-13: order-independent merge. Highest confidence wins. Same confidence → deterministic (first encountered wins). Cannot weaken detection via locale ordering. |
| T-04-02-04 | Denial of Service | Multi-locale with 1000 codes | mitigate | D-114: comma-separated list. Cap at 10 entries per LOCL-05. Extra entries truncated with logged warning. |
| T-04-02-05 | Elevation of Privilege | Universal recognizer weakening | accept | AG-14 (cross-plan): presets cannot weaken security. Locale negotiation adds locale-specific recognizers but does not override or disable universal recognizers — merge is always additive. |
</threat_model>
