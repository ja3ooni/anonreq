<objective>
Implement the locale recognizer bundle system for the AnonReq gateway. This plan delivers: (1) LocaleBundle Pydantic model with entity types, recognizer tiers, confidence thresholds, and checksum config, (2) 8 locale YAML files in config/locales/ for en, de-DE, fr-FR, nl-NL, es, it-IT, ar, pt-BR, (3) LocaleRegistry that auto-discovers bundles at startup, (4) ChecksumValidator framework with ABC and 4 concrete validators (ISO7064 Mod 11,2 for Steuer-ID, Luhn for BSN/CPF/CNPJ, NIR for French social security, Codice Fiscale for Italian tax ID), (5) ChecksumValidatorRegistry keyed by entity_type, (6) integration hook in DetectionEngine for post-detection checksum validation, and (7) property-based test proving invalid checksum IDs never appear as valid detections.

Purpose: Without locale bundles, the gateway only detects universal PII patterns (English-centric). All 8 locale detection requirements (LOCL-01, LOCL-06, LOCL-07) depend on this plan's data and framework.

Output: A working `LocaleRegistry` that discovers 8 YAML bundles at startup, and `ChecksumValidatorRegistry` that validates national ID checksums, with property-tested proof that invalid checksums never produce detections.
</objective>

    2. **src/anonreq/locale/checksum.py**:
       - `class ChecksumValidator(ABC)`:
         - `@abstractmethod def validate(self, digits: str) -> bool: ...`
       - `class ChecksumValidatorRegistry`:
         - `_validators: dict[str, ChecksumValidator] = {}`
         - `def register(self, entity_type: str, validator: ChecksumValidator) -> None`
         - `def get(self, entity_type: str) -> ChecksumValidator | None`
         - `def validate(self, entity_type: str, value: str) -> bool`
       - `def validate_detection(detection: dict, registry: ChecksumValidatorRegistry) -> dict | None`:
         - If detection.entity_type has registered validator:
           - If validator.validate(extracted_digits): return detection (keep)
           - Else: return None (drop)
         - Else: return detection (pass-through)

    3. **src/anonreq/locale/checksums/iso7064.py**:
       - `class ISO7064Mod11_2Validator(ChecksumValidator)`:
         - `def validate(self, digits: str) -> bool`:
           - Strip whitespace, assert length 11
           - Apply ISO 7064 Mod 11,2 algorithm:
             - product = 10
             - For each digit (0-10 positions): product = (product + int(digit)) % 11 or 10; if product == 0: product = 10; product = (product * 2) % 11
             - Check digit = 11 - product (if == 10 → invalid)
             - Compare with last digit
           - Return bool

    4. **src/anonreq/locale/checksums/luhn.py**:
       - `class LuhnValidator(ChecksumValidator)`:
         - `def validate(self, digits: str) -> bool`:
           - Strip non-digits, validate length
           - Standard Luhn algorithm: double every second digit from right, sum digits, check total % 10 == 0
       - `class CPFValidator(ChecksumValidator)`:
         - CPF-specific: 11 digits, two check digits with modulo 11
       - `class CNPJValidator(ChecksumValidator)`:
         - CNPJ-specific: 14 digits, two check digits with modulo 11

    5. **src/anonreq/locale/checksums/nir.py**:
       - `class NIRValidator(ChecksumValidator)`:
         - `def validate(self, digits: str) -> bool`:
           - French NIR: 15 digits (13 + 2 key digits)
           - Strip whitespace
           - Validate gender digit (1 or 2)
           - Validate birth month (01-12)
           - Validate key: 97 - (NIR_without_key % 97) == key
           - Return bool

    6. **src/anonreq/locale/checksums/codice_fiscale.py**:
       - `class CodiceFiscaleValidator(ChecksumValidator)`:
         - `def validate(self, code: str) -> bool`:
           - Italian Codice Fiscale: 16 characters
           - Validate check character at position 16 using Italian algorithm:
             - Sum odd-position characters (letter→position in alphabet, digit→its value)
             - Sum even-position characters (letter→position in alphabet, digit→digit sum)
             - Total % 26 → check character (A=0, B=1, ... Z=25)
           - Compare with actual check character
