<objective>
Implement streaming detection with sliding window buffer and the admin API for firewall rules. This plan delivers: (1) SlidingWindowDetector — buffers ~2KB of streaming content and runs detection at chunk boundaries; (2) Hot-reload mechanism — atomically replaces rule set within 60 seconds without restart; (3) Admin endpoint GET /v1/admin/prompt-security/rules — listing active rules with metadata; (4) Streaming detection integration with outbound gates.

Purpose: Without streaming detection, injection in streaming responses would go undetected. Without hot-reload, security teams cannot respond to emerging threats without restarting the gateway.
</objective>

    1. **src/anonreq/firewall/streaming.py**:
       - `SlidingWindowDetector` class:
         - Constructor: `__init__(self, engine: FirewallRuleEngine, window_size: int = 2048)`
         - Field: `_buffer: str = ""`, `_window_size: int`
         - `async def ingest(self, chunk: str) -> list[DetectionResult]`:
           - Append chunk to buffer
           - If buffer exceeds window_size: discard oldest bytes to fit
           - Run engine.evaluate() on current buffer
           - Return detection results
         - `async def flush(self) -> list[DetectionResult]`:
           - Run detection on remaining buffer content
           - Clear buffer
           - Return any final detection results
         - `def reset(self) -> None`:
           - Clear buffer without detection
       - `StreamingFirewallDetector` class:
         - Wraps SlidingWindowDetector for outbound streaming pipeline
         - `async def process_chunk(self, chunk: bytes, ctx: ProcessingContext) -> tuple[bytes, list[DetectionResult]]`:
           - Decode chunk, pass to SlidingWindowDetector.ingest()
           - If detected: apply action (BLOCK/FLAG/MONITOR)
           - Return (original_chunk_or_replacement, detection_results)

    Implement the hot-reload mechanism.

    1. **src/anonreq/firewall/reloader.py**:
       - `FirewallRuleReloader` class:
         - Constructor: `__init__(self, loader: FirewallRuleLoader)`
         - `async def reload(self) -> tuple[list[FirewallRule], list[FirewallRule]]`:
           - Call loader.reload() to get new rules
           - Atomic swap: acquire lock, replace rules list, release lock
           - Return (old_rules, new_rules)
         - `async def start_watcher(self, interval: int = 30)`:
           - Background task checking file mtime every `interval` seconds
           - If mtime changed: call reload()
           - On reload success: emit `firewall_rule_reloaded` audit event
           - On reload failure: log error, keep existing rules
         - `async def stop_watcher(self) -> None`:
           - Cancel background watcher task
         - Uses `asyncio.create_task` for background watcher
         - Uses `threading.Lock` or `asyncio.Lock` for atomic swap

    2. **tests/firewall/test_reloader.py**:

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Streaming chunks → SlidingWindow | Partial content buffered for detection |
| File system → Reloader | Rule file modifications trigger atomic reload |
| Admin API → Rule list | Rule metadata exposed to authorized users |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-03-01 | Denial of Service | Sliding window memory | mitigate | Bounded window (2KB max); older content discarded |
| T-10-03-02 | Tampering | Hot-reload race condition | mitigate | Lock-protected atomic swap; concurrent reads get consistent old or new set |
| T-10-03-03 | Tampering | Rule file tampering | mitigate | File watcher detects changes; invalid file rejected (existing rules preserved) |
| T-10-03-04 | Information Disclosure | Rule patterns in API response | mitigate | Patterns are metadata-only (regex strings); no raw data exposure; endpoint requires auth |
</threat_model>
