<objective>
Build the AI Security Firewall rule engine and ML model foundation. This plan delivers: (1) Firewall Pydantic models for rules, detection categories, severity, and actions; (2) YAML rule loader with semantic rules and regex patterns for all 7 attack categories; (3) Rule evaluation engine with configurable per-category thresholds; (4) ONNX ML model integration for deep analysis on flagged requests; (5) Default rule set covering prompt injection, jailbreak, system prompt extraction, instruction override, role escalation, hidden tool invocation, and secret exfiltration.

Purpose: Without the rule engine and ML model, there is no detection capability. Every firewall gate (inbound/outbound/streaming) depends on this foundation.
</objective>

    1. **src/anonreq/firewall/engine.py**:
       - `FirewallRuleEngine` class:
         - Constructor: `__init__(self, rules: list[FirewallRule], category_config: dict[str, RuleCategoryConfig], severity_mapping: SeverityActionMapping)`
         - `async def evaluate(self, text: str) -> list[DetectionResult]`:
           - For each enabled rule in priority order:
             - If rule has pattern: regex match against text
             - If match found AND confidence >= category threshold: record DetectionResult
             - If rule has no pattern (semantic-only): mark for ML evaluation
           - Group by category, return highest severity result per category
           - Respect latency budget: stop evaluation if budget exceeded
         - `async def evaluate_with_ml(self, text: str, ml_model: MLModel) -> list[DetectionResult]`:
           - Run rules first (fast path)
           - If any rule flagged: run ML model for deep analysis
           - Merge results: ML confidence may override rule confidence
         - `def _match_rule(self, rule: FirewallRule, text: str) -> tuple[bool, float]`:
           - If pattern present: re.compile(rule.pattern).search(text)
           - Return (matched, confidence) where confidence = 1.0 for exact pattern match, or partial match score
         - `def _get_highest_severity(self, results: list[DetectionResult]) -> DetectionResult`:
           - Return result with highest severity (HIGH > MEDIUM > LOW)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML rule file → Rule loader | Untrusted rule definitions parsed at startup/reload |
| User prompt → Rule engine | Untrusted text enters detection pipeline |
| User prompt → ML model | Untrusted text passed to ML inference |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01-01 | Tampering | YAML rule injection | mitigate | `yaml.safe_load()` prevents code execution; Pydantic validation rejects malformed rules |
| T-10-01-02 | Information Disclosure | ML model output | mitigate | Model returns confidence scores and category only; no raw text in model output |
| T-10-01-03 | Denial of Service | ML inference timeout | mitigate | Configurable inference timeout (default 200ms); fallback to rules-only on timeout |
| T-10-01-04 | Tampering | Rule priority bypass | mitigate | Rules evaluated in priority order; highest priority match determines result |
| T-10-01-05 | Information Disclosure | matched_text_snippet in audit | mitigate | Snippet limited to 50 chars; contains match context, not full prompt/response |
</threat_model>
