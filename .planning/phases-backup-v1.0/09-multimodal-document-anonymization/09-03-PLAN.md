<objective>
Extend the Restore Engine with path-aware token tracking for structured content types, and implement local routing for unsupported content types. This plan delivers: (1) PathTracker — records JSONPointer/dot-notation paths for each token during anonymization, enabling accurate restoration in structured documents; (2) Extended Restore Engine — path-aware replacement for JSON/multipart/tool calls + streaming-aware restoration via Tail_Buffer for split tokens; (3) LocalRouter — routes unsupported content types to on-prem endpoints instead of forwarding; (4) Payload validation integration with pipeline.

Purpose: Without path-aware restoration, tokens in JSON documents and tool calls cannot be accurately restored, breaking the round-trip guarantee. Unsupported content types must be safely routed locally rather than crashing or being silently dropped.
</objective>
