  <name>Task 1: Implement Hypothesis property-based tests for round-trip correctness</name>
  <files>
    tests/multimodal/test_property.py
  </files>
  <action>
    Implement Hypothesis property-based tests for all multimodal content types.

    1. **tests/multimodal/test_property.py**:
       - Import Hypothesis: `from hypothesis import given, assume, strategies as st, settings`
       - Define strategies:
         - `json_value`: recursive strategy generating valid JSON values (dicts, lists, strings, numbers, booleans, null) with at least one detectable PII string
         - `pii_string`: generates strings containing email, phone, SSN, or credit card patterns
         - `multipart_parts`: generates list of (name, content_type, body) tuples
         - `content_type_strategy`: `st.sampled_from(["text/plain", "application/json", "multipart/form-data"])`
         - `tool_call_format`: `st.sampled_from(["openai", "anthropic", "mcp"])`

       - Property tests:
         - `test_round_trip_text(generated_text)`:
           - For any text with detectable PII: anonymize → restore → byte-for-byte match
         - `test_round_trip_json(generated_json)`:
           - For any JSON structure with PII in string leaves: anonymize → restore → byte-for-byte match
         - `test_round_trip_multipart(generated_multipart)`:
           - For any multipart payload: anonymize → restore → byte-for-byte match
         - `test_json_structure_preserved(generated_json)`:
           - After anonymization, resulting JSON is parseable with identical structure
           - Only string leaf values containing PII are modified
           - Keys, numbers, booleans, nulls, and non-PII strings unchanged
           - Compare JSON structural fingerprint (key set, types, nesting depth, array lengths)
         - `test_no_raw_pii_after_anonymize(generated_payload)`:
           - After anonymization, no detectable PII patterns remain in string values
           - Use same regex patterns as detection engine for verification
         - `test_token_collisions_across_sessions(payload1, payload2)`:
           - Same PII value in separate sessions produces different tokens
           - No token collision across 100+ generated session pairs
         - `test_streaming_round_trip(generated_json, split_position)`:
           - For every possible token split position (including token boundaries):
           - Anonymize → stream with Tail_Buffer → restore → byte-for-byte match
         - `test_tool_call_round_trip(generated_tool_call)`:
           - For OpenAI/Anthropic/MCP format: anonymize → restore → arguments match

    2. Run with `pytest tests/multimodal/test_property.py -x --tb=short -v --hypothesis-show-statistics`
  </action>
