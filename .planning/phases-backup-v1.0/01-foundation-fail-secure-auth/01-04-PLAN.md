    2. Add a middleware that populates request_id on every request BEFORE auth runs:
       ```python
       @app.middleware("http")
       async def set_request_id(request: Request, call_next):
           request_id = f"req_{uuid4().hex[:24]}"
           request.state.request_id = request_id
           request.state.context = RequestContext(request_id=request_id)
           structlog.contextvars.bind_contextvars(request_id=request_id)
           response = await call_next(request)
           structlog.contextvars.unbind_contextvars("request_id")
           return response
       ```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04-01 | Spoofing | verify_api_key | mitigate | HTTPBearer validates Authorization header format. Token compared against settings.ANONREQ_API_KEY. Wrong/missing token returns 401 before any route handler runs per D-01. |
| T-01-04-02 | Information Disclosure | 401 error response | mitigate | Auth errors return OpenAI-compatible envelope (not FastAPI default `{"detail": "Not authenticated"}`) — no stack traces, no hints about key length or format beyond "Invalid API key" per RESEARCH Pitfall 3. |
| T-01-04-03 | Information Disclosure | request_id in error responses | accept | request_id is a random hex string with no entropy on env vars or request data. It's safe to include in error responses for trace correlation. |
| T-01-04-04 | Denial of Service | RequestContext UUID allocation | accept | uuid4() is lightweight (~60ns per call). No DoS concern even under high load. |
| T-01-04-SC | Tampering | No new packages in this plan | accept | Plan 01-04 only uses stdlib modules (uuid, dataclasses) and existing dependencies (FastAPI HTTPBearer). No new package installs. |
