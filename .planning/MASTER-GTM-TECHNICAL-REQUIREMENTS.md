# AnonReq GTM Technical Requirements

## Buyer-Visible Proof

AnonReq must demonstrate no raw sensitive data leaves the perimeter, no PII appears in logs, tenant isolation holds under concurrency, provider choice works through one OpenAI-compatible interface, and compliance evidence can be exported without professional services.

## Pilot Requirements

Pilot deployments must support Docker Compose in under 30 minutes, Kubernetes Helm for enterprise evaluation, OIDC or API key authentication, GDPR preset, at least English and German locale bundles, OpenAI and Anthropic providers, metadata audit logs, `/health`, `/metrics`, and a no-PII log verification script.

## Enterprise Procurement Requirements

Security questionnaires require SBOM, SECURITY.md, incident response, architecture diagrams, data-flow diagrams, threat model, vulnerability process, SOC2/ISO readiness mapping, DORA/NIS2/GDPR mapping, SLO runbooks, retention/legal hold design, and signed release artifacts.

## Commercial Differentiators

Enterprise and appliance tiers add SSO/RBAC, policy engine, admin portal, advanced DLP, AI firewall, SIEM/CASB/RAG/voice connectors, Trust Center, governance reporting, and regulated-industry evidence packages.
