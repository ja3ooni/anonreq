<objective>
Build the Content-Type Dispatcher middleware and the two core content analyzers (JSON and Multipart) for the multimodal document anonymization pipeline. This plan delivers: (1) Content-Type Dispatcher middleware that routes requests to the correct analyzer after PDP #1 and before PDP #2; (2) JSON Analyzer with recursive tree walk and sensitive key-pattern detection; (3) Multipart Analyzer that parses form-data and routes parts by content type; (4) Unified Detection Result structure consumed by the Phase 2 Anonymization Engine; and (5) Payload size/depth limit validation.

Purpose: Without this dispatcher and analyzers, the gateway can only handle text/plain requests. Every multimodal and structured content type must be routed, analyzed, and validated before reaching the anonymization engine.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Content-Type header → Dispatcher | Untrusted header controls routing decision |
| JSON body → JsonAnalyzer | User-supplied JSON parsed and traversed |
| Multipart body → MultipartAnalyzer | User-supplied multipart data parsed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01-01 | Tampering | Content-Type header bypass | mitigate | Unknown types → ROUTE_LOCAL (never FORWARD); explicit allowlist |
| T-09-01-02 | Denial of Service | Deeply nested JSON | mitigate | max_depth=50 enforced; depth exceeded → ROUTE_LOCAL, not crash |
| T-09-01-03 | Denial of Service | Oversized payload | mitigate | Size limits enforced before processing; exceeded → BLOCK/ROUTE_LOCAL |
| T-09-01-04 | Denial of Service | Multipart part explosion | mitigate | max_parts=100 limit; excessive parts rejected early |
| T-09-01-05 | Information Disclosure | Sensitive key boosting | mitigate | Key detection only; confidence boost applied to existing detection — no new entity creation |
</threat_model>
