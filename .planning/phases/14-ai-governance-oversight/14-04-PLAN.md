    1. **tests/property/test_governance_versioning.py** — per TEST-04/TEST-05 equivalent:
       - **Append-only invariant**: For any sequence of create_version and rollback_version calls:
         - Number of versions always equals number of create calls (no overwrites)
         - Each version has unique (object_type, object_id, version_number) tuple
         - No version's snapshot changes after creation
         - Version numbers are strictly increasing (1, 2, 3, ...) with no gaps
       - **Rollback correctness invariant**: After rollback to version N, the current snapshot equals version N's snapshot, AND a new version N+1 exists with that snapshot
       - **Diff consistency invariant**: Applying compute_diff(v1, v2) to v1's snapshot produces v2's snapshot exactly
       - Use Hypothesis `strategies.text()`, `strategies.dictionaries()`, and `strategists.lists()` to generate arbitrary governance objects and operation sequences

    2. **tests/property/test_governance_lifecycle.py**:
       - **Stage machine invariant**: For any sequence of transition calls, the system never enters an invalid stage given the previous stage. Verified against VALID_TRANSITIONS.

       - **Gate invariant**: APPROVED→PRODUCTION always fails unless all 3 gates pass.
