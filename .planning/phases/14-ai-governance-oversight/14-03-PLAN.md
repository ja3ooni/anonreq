    1. **Add to src/anonreq/models/governance.py**:
       - `LifecycleStage` enum: `DRAFT`, `REVIEW`, `APPROVED`, `PRODUCTION`, `DEPRECATED`, `RETIRED`
       - `LifecycleObject(BaseModel)`: id (str), object_type (str), object_id (str), current_stage (LifecycleStage), stage_history (list[dict]), approval_required (bool=True), gates (dict | None)

    2. **src/anonreq/governance/lifecycle.py**:
       - `STAGES = ["DRAFT", "REVIEW", "APPROVED", "PRODUCTION", "DEPRECATED", "RETIRED"]`
       - `VALID_TRANSITIONS = { "DRAFT": ["REVIEW"], "REVIEW": ["APPROVED", "DRAFT"], "APPROVED": ["PRODUCTION", "DRAFT", "DEPRECATED"], "PRODUCTION": ["DEPRECATED"], "DEPRECATED": ["RETIRED", "PRODUCTION"], "RETIRED": [] }`
       - `class LifecycleManager`:
         - `__init__(self, db: AsyncSession, approval_service, risk_service)`
         - `async def get_current_stage(self, object_type: str, object_id: str) -> LifecycleStage`
         - `async def get_stage_history(self, object_type: str, object_id: str) -> list[dict]`
         - `async def transition(self, object_type: str, object_id: str, to_stage: str, operator: str, approval_id: int | None = None) -> LifecycleStage`:
           - Validates transition is in VALID_TRANSITIONS[current_stage]
           - If to_stage in ["REVIEW", "APPROVED"]: requires approval_id (creates approval request if not provided)
           - If to_stage == "PRODUCTION": checks production gates:
             - `gates_passed()`: testing_complete=True AND risk_assessment_done=True AND governance_approval_id IS NOT None
           - Records transition in stage_history: `{"from": current, "to": to_stage, "at": now, "by": operator, "approval_id": approval_id}`
           - Updates current_stage
           - Emits `lifecycle_transition` audit event
       - `async def check_production_gates(self, object_type: str, object_id: str) -> dict`:
         - Returns `{"passed": bool, "gates": {"testing_complete": bool, "risk_assessment_done": bool, "governance_approval": bool}}`

    3. **Add to router.py**:
       - `GET /v1/governance/lifecycle/{object_type}/{object_id}` — get current stage + history
       - `POST /v1/governance/lifecycle/{object_type}/{object_id}/transition` — request transition
         - Body: `{"to_stage": "PRODUCTION", "operator": "admin"}`
       - `GET /v1/governance/lifecycle/{object_type}/{object_id}/gates` — check production gates

    4. **tests/test_governance_lifecycle.py** — full transition matrix tests.
  </action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Lifecycle API → PostgreSQL | Stage transitions persisted |
| Transparency middleware → Response headers | Metadata injected into outbound response |
| Conformity package → ZIP file → Download | Assembled compliance data served to admin |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-03-01 | Tampering | Lifecycle stage transitions | mitigate | Approval gate requirement prevents unauthorized stage advancement |
| T-14-03-02 | Information Disclosure | Transparency records | mitigate | Metadata only — no raw request/response content per D-026 |
| T-14-03-03 | Information Disclosure | Conformity package | mitigate | Admin auth required; package contains only compliance data already accessible via other admin endpoints |
| T-14-03-04 | Tampering | Conformity package | mitigate | Generated on-demand from current state; release snapshot is immutable artifact |
| T-14-03-05 | Elevation of Privilege | Production gate bypass | mitigate | All 3 gates (testing, risk, approval) must pass; server-side check not bypassable |

</threat_model>
