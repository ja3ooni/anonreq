<objective>
Close the appliance reverse/transparent proxy anonymization blocker by adapting proxy byte dispatch into the existing runtime pipeline contract.

Purpose: ReverseProxy and TransparentProxy already call content_dispatcher.dispatch(...), but main.py currently supplies app.state.pipeline, which exposes run(ctx) rather than dispatch(...). This plan adds a narrow adapter so proxy AI traffic enters the same anonymization, DLP, tool governance, forwarding guard, provider, restoration, and cleanup path as chat requests.
Output: A proxy dispatcher adapter, deployment proxy wiring, and tests proving proxy paths do not echo raw request bodies or status-only routed responses.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| appliance proxy to dispatcher | Raw intercepted AI traffic enters AnonReq outside the normal FastAPI chat route. |
| dispatcher to provider stage | Sanitized proxy payload may leave the enterprise boundary if all controls pass. |
| dispatcher to proxy response | Restored or error response returns to the original client. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-03-01 | Information Disclosure | PipelineContentDispatcher | mitigate | Parse proxy AI requests into ProcessingContext and run PipelineManager before provider forwarding. |
| T-22-03-02 | Tampering | malformed proxy payloads | mitigate | Reject invalid JSON and unsupported content types fail-closed with structured safe error bytes. |
| T-22-03-03 | Spoofing | tenant/request context from proxy ctx | mitigate | Treat ctx metadata as untrusted hints and default tenant/request ids safely when absent. |
| T-22-03-04 | Information Disclosure | proxy error paths | mitigate | Tests assert raw request body and synthetic PII are absent from adapter error responses. |
| T-22-03-SC | Tampering | package installs | mitigate | No package-manager installs are allowed in this plan. |
</threat_model>
