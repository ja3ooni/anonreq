<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client chat request to pipeline | Untrusted prompt, tool calls, tool arguments, and metadata enter runtime enforcement. |
| provider response to gateway | Untrusted model output returns before restoration and client delivery. |
| governance audit emission | Security metadata enters audit/log sinks and must remain raw-content-free. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-02-01 | Information Disclosure | InboundDLPStage | mitigate | Run DLP before ForwardingGuard/ProviderStage and assert blocked inputs do not reach provider spies. |
| T-22-02-02 | Information Disclosure | OutboundDLPStage | mitigate | Inspect provider_response before RestorationStage/client delivery and abort on blocked outbound content. |
| T-22-02-03 | Elevation of Privilege | ToolGovernanceStage | mitigate | Evaluate tool calls with PDPToolEvaluator and block cross-domain or disallowed tools before provider forwarding. |
| T-22-02-04 | Repudiation | Tool audit metadata | mitigate | Emit metadata-only tool decisions with event types for allowed, blocked, and approval-required outcomes. |
| T-22-02-05 | Denial of Service | Missing DLP/governance components | mitigate | Fail closed when enabled enforcement services are required but unavailable. |
| T-22-02-SC | Tampering | package installs | mitigate | No package-manager installs are allowed in this plan. |
</threat_model>
