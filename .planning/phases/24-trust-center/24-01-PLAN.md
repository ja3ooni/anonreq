<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public internet → Trust Center API | Untrusted external requests access aggregate metadata endpoints. No authentication. Rate-limited. |
| Trust Center service → SLO Engine | Public endpoint reads internal SLO compliance data via in-process call. |
| Trust Center service → PresetEngine | Public endpoint reads internal compliance registry via in-process call. |
| Trust Center service → prometheus REGISTRY | Public endpoint reads aggregate Prometheus counters from in-process registry. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01 | Information Disclosure | Metrics endpoint | mitigate | Rate limiting (60 RPM per IP) limits scraping velocity. Documentation warns that aggregate request volumes can leak business intelligence. No per-tenant or per-session breakdowns returned. |
| T-24-02 | Denial of Service | SLO Engine / PresetEngine backends | mitigate | Rate limiting (60 RPM per IP) caps ingress. Fail-closed pattern (service layer catches exceptions, returns 503 fast) prevents cascading load on SLO engine. |
| T-24-03 | Information Disclosure | Error responses | mitigate | Service layer catches all exceptions from SLO/PresetEngine dependencies, logs the error internally, returns generic `{"error": "service_unavailable"}` 503. No internal state or stack trace exposed. |
| T-24-04 | Tampering | Config gate bypass | mitigate | Enable gate runs as `Depends()` on every request — checks `app.state.trust_center_enabled` at runtime. Cannot be bypassed by request manipulation. |
| T-24-05 | Spoofing | Rate limiter IP tracking | accept | Rate limiter uses `request.client.host`. In proxied deployments, this reflects the reverse proxy IP, not the end client. For trusted-internal-network deployments this is acceptable per SPEC §2.2. Production reverse proxy should set `X-Forwarded-For` and configure middleware to use it. |
| T-24-06 | Repudiation | Public endpoint access | accept | Trust Center is a public metadata portal. No audit logging for individual requests per CONTEXT.md risk notes and SPEC metadata-only constraint. |
| T-24-SC | Tampering | npm/pip/cargo installs | accept | No new external dependencies added in this phase. All imports use existing pyproject.toml deps (fastapi, pydantic-settings, pyyaml, prometheus-client, fakeredis). |
</threat_model>
