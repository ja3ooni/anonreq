<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cache command -> retry scheduler | Transient service failure may be retried, but arbitrary failures must never be hidden or retried indefinitely. |
| Cache availability -> anonymization pipeline | A missing or unreadable mapping can permit an unsafe forwarding/restoration path unless terminal failure returns HTTP 503 before ProviderStage. |
| Orchestrator probes -> FastAPI routes | Liveness controls restart behavior; readiness controls ingress traffic admission during a reelection. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|----------|----------|-----------|-------------|-----------------|
| T-28-02 | Information Disclosure / Elevation of Privilege | cache operation error path | mitigate | Narrowly retry transient errors, then preserve the established metadata-safe 503 dependency error through tokenization and prove ProviderStage is not called. |
| T-28-03 | Denial of Service | Sentinel/Cluster reelection | mitigate | Use the locked 30-second exponential retry with jitter to constrain retries and dampen reconnect bursts. |
| T-28-04 | Information Disclosure | health/startup diagnostics | mitigate | Return component statuses only; suppress URLs, cache contents, and backend exception text. |
| T-28-05 | Denial of Service | orchestration probe policy | mitigate | Keep liveness process-only so normal reelection does not create a restart loop; make readiness 503 so ingress withdraws the instance. |
</threat_model>
