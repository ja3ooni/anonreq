<objective>
Wire all Phase 2 components into a working pipeline with `POST /v1/chat/completions` as the entry point. This plan delivers: (1) PipelineStage ABC + PipelineManager with stage registry and sequential execution, (2) individual pipeline stages that wire the engines built in 02-02 and 02-03, (3) ForwardingGuard — the fail-secure gate before any outbound call, (4) ProviderStage — async HTTP passthrough to OpenAI, (5) RestorationStage — token→value replacement in responses via Restorer, (6) CleanupStage — async DEL + audit logging, (7) the POST /v1/chat/completions route handler, and (8) full pipeline integration tests.

Purpose: This is the orchestration layer that ties all engines together into a working request→response flow. Without this plan, the individual engines exist but no request can be processed end-to-end.

Output: A working `POST /v1/chat/completions` that accepts a request, runs it through the full pipeline, and returns the anonymized response. All HTTP status codes (200, 403, 500, 503) handled correctly per fail-secure requirements.
</objective>
