Output: a secret-source abstraction, startup bootstrap wiring, and tests proving the secret path is in-memory only.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Secret backend -> startup bootstrap | External secret data becomes trusted runtime configuration only after validation. |
| Runtime secret store -> provider adapters | Provider code must consume memory-only secrets without re-exposing them. |
| Startup bootstrap -> logs/config | Secret values and backend connection details must not leak to disk, env, or logs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|----------|----------|-----------|-------------|-----------------|
| T-29-01 | Information Disclosure | secret bootstrap path | mitigate | Keep credentials in an in-memory store and never emit values or source details to logs. |
| T-29-02 | Tampering | provider credential resolution | mitigate | Require backend validation before the secret store is populated. |

</threat_model>
