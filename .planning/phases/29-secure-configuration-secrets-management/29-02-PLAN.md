<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Secret/config volume -> in-memory snapshot | File changes become trusted runtime state only after validation and atomic swap. |
| Watcher -> runtime store | Reload logic must not leave a partially written snapshot visible to callers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|----------|----------|-----------|-------------|-----------------|
| T-29-03 | Tampering | reload path | mitigate | Validate then swap atomically; keep the previous snapshot until replacement succeeds. |
| T-29-04 | Information Disclosure | hot reload logs | mitigate | Avoid logging secret paths, secret values, or backend exception text. |

</threat_model>
