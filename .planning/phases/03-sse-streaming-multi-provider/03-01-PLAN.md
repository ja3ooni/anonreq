<objective>
Implement SSE streaming support for the AnonReq gateway. This plan delivers: (1) the StreamEvent canonical model for normalizing all provider stream types, (2) the TailBuffer FSM that reassembles chunks and never emits partial token matches, (3) the streaming RestorationStage that replaces tokens with case-insensitive + bracket-optional matching via pre-fetched Valkey mappings, (4) the SSEEmitter that formats text/event-stream with anti-buffering headers, (5) the streaming route handler that branches on stream:true, wires stages through TailBuffer → RestorationStage → Emitter, (6) client disconnect handling with upstream HTTPX cancellation and idempotent SessionCleanup, and (7) TTL extension for long-running streams at 80% elapsed time.

Purpose: Without this plan, the gateway cannot handle streaming LLM responses — all requests would be forced to non-streaming mode. SSE-01 through SSE-08, CACH-05, and PROV-02 through PROV-04 all depend on this streaming infrastructure.

Output: A working `text/event-stream` streaming response for `POST /v1/chat/completions` with `stream: true`, tokens restored in real-time with correct partial-token handling, and clean disconnect handling.
</objective>

    1. **src/anonreq/streaming/tail_buffer.py**:
       - `class BufferState(str, Enum)`: COLLECTING, MATCHING, FLUSHING, TERMINATED
       - `class TailBuffer`:
         - Fields: active_buffer (str), tail_window (str), state (BufferState), last_flush_at (float)
         - Constants: TAIL_WINDOW_CHARS=128, MAX_BUFFER_CHARS=2048, MAX_BUFFER_AGE_MS=1000
         - `async def ingest(self, event: StreamEvent) -> AsyncIterator[str]`:
           - Only process TEXT_DELTA events — all other event types are yielded as-is (bypass FSM)
           - Append `event.delta_text` to `self.active_buffer`
           - Transition to MATCHING state
         - `async def _match(self) -> AsyncIterator[str]`:
           - Scan active_buffer for `[TYPE_N]` pattern at or near buffer tail (within TAIL_WINDOW_CHARS)
           - Full token match at buffer frontier → transition to FLUSHING
           - Partial match at tail (start of `[` but incomplete) → retain in buffer, transition to COLLECTING (wait for more data)
           - No match found → transition to FLUSHING
         - `async def _flush(self) -> AsyncIterator[str]`:
           - Determine safe_prefix: content before the last TAIL_WINDOW_CHARS
           - If safe_prefix: emit safe_prefix (this is the assembled text chunk)
           - Retain tail_window = last TAIL_WINDOW_CHARS of buffer for next round
           - Set last_flush_at = time.monotonic()
           - Transition to COLLECTING (or TERMINATED if finish event received)
         - `def terminate(self) -> None`: set state to TERMINATED
         - `def flush_remaining(self) -> str`: flush entire buffer including tail window — called on FINISH event
         - Age-based flush check: called before each MATCHING transition, emits if buffer_age > MAX_BUFFER_AGE_MS
         - Size-based flush check: called before each MATCHING transition, flushes if len(active_buffer) > MAX_BUFFER_CHARS
       - Import `re` for token pattern matching. Use the `TOKEN_PATTERN` constant: `r'\[[A-Z][A-Z_]{0,19}_\d+\]'`
       - Use `time.monotonic()` for age checks — not `time.time()` (monotonic is immune to system clock changes)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Gateway → Client (SSE stream) | Restored token values stream back to client over HTTP |
| Gateway → Upstream Provider | Provider API stream flows through TailBuffer for reassembly |
| ASGI Disconnect Signal | External client disconnect triggers upstream cancellation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Information Disclosure | Streaming RestorationStage | mitigate | HGETALL pre-fetch loads mappings into memory for stream duration. Released in close_session() post-stream. Mapping never written to logs per AUDT-02. |
| T-03-01-02 | Tampering | TailBuffer partial token match | mitigate | AG-05: TailBuffer never emits partial token matches. Buffer retains tail window (128 chars) until full match or flush trigger. D-65: No chunk-count-based flushing — chunk count is provider-dependent, no semantic meaning. |
| T-03-01-03 | Denial of Service | TailBuffer unbounded growth | mitigate | MAX_BUFFER_CHARS=2048 hard limit. Buffer flushed before exceeding this ceiling. Tail window capped at TAIL_WINDOW_CHARS=128. |
| T-03-01-04 | Tampering | SessionCleanup race condition | mitigate | _cleaned flag ensures exactly-once execution per D-87. First terminal state wins — FINISH vs DISCONNECT race handled by checking _cleaned before each cleanup step. |
