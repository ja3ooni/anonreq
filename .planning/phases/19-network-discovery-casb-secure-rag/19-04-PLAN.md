<objective>
Build the CASB (Cloud Access Security Broker) policy engine for AI SaaS governance. This wave delivers: (1) Phase 8 policy YAML extension with `apps:` section for CASB app classification (sanctioned/tolerated/unsanctioned), (2) CASB enforcement intercept that applies per-app policy actions to matching requests, (3) user group resolution from request context with configurable identity chain, (4) per-group policy overrides, (5) CASB audit events and Prometheus metrics, and (6) `GET /v1/admin/casb/activity` endpoint for querying CASB activity.

Purpose: Without CASB controls, enterprises have no way to govern which AI SaaS applications employees use. CASB extends proxy-level governance to application-level visibility and enforcement.

Output: AI SaaS application requests classified and governed — sanctioned apps allowed, unsanctioned apps blocked with HTTP 451, tolerated apps alerted.
</objective>

  <action>
    Implement CASB enforcement and observability per D-009, D-010, D-011.

    1. **src/anonreq/casb/enforcer.py:**
       - `CASBEnforcer` class:
         - Constructor takes `policy_loader: CASBPolicyLoader`, `user_resolver: UserResolver`, `hostname_matcher: HostnameMatcher`, `audit_logger`.
         - `async enforce(request: Request) -> CASBDecision | None`:
           1. **Identify app**: Extract target hostname from request. Match against known AI SaaS apps using `hostname_matcher`.
           2. **Resolve user**: `user_identity = user_resolver.resolve(request)`.
           3. **Look up policy**: Find app in CASB app config. If not found → return None (no CASB action).
           4. **Resolve action**: `action = policy_loader.resolve_action(app_id, user_identity.groups)`.
           5. **Enforce**:
              - If `ALLOW`: create decision(allow=true), optionally audit.
              - If `ALERT`: create decision(allow=true, alert=true), emit audit event, add `X-AnonReq-Warning: CASB tolerated app {app_id}` header.
              - If `BLOCK`: create decision(allow=false), emit audit event, return HTTP 451 response.
           6. Return `CASBDecision(allow, action, app_id, user_identity)`.
         - `CASBDecision` dataclass:
           - `allowed: bool`, `action: AppAction`, `app_id: str`.
           - `user_id: str | None`, `tenant_id: str`.
           - `response_headers: dict | None`.

    2. **src/anonreq/casb/audit.py:**
       - `emit_casb_event(audit_logger, app_id, classification, action, user_id, tenant_id)`:
         - Emits `event_type: unsanctioned_ai_access`.
         - Fields: application, classification, action_taken, user_id, tenant_id, timestamp.
       - Prometheus metrics:
         ```python
         CASB_EVENTS = Counter(
             "anonreq_casb_events_total",
             "CASB policy enforcement events",
             ["application", "classification", "action"]
         )
         ```

         - Requires admin auth.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Request → CASB Enforcer | Request headers inspected for app identification |
| Header/JWT → UserResolver | User identity extracted from headers or tokens |
| Admin API → Activity | CASB activity log exposed via admin API |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-04-01 | Spoofing | User identity | mitigate | Resolver chain supports multiple auth methods. JWT signature validation prevents forged identities. Header-only mode is trust-based (assumes upstream proxy). |
| T-19-04-02 | Tampering | CASB policy YAML | mitigate | Phase 8 YAML loading with schema validation. CASBPolicyLoader.validate() rejects unknown classification values. |
| T-19-04-03 | Bypass | App identification | mitigate | CASB enforcement uses HostnameMatcher (Phase 19-01) which matches hostname + IP. Direct-IP access to unsanctioned apps still blocked by IP matching. |
| T-19-04-04 | Information Disclosure | CASB activity | mitigate | Admin API requires admin auth. Activity log contains metadata only — no request body, no response body. |
</threat_model>
