<objective>
Build the Secure RAG retrieval pipeline with the Retrieval Policy Engine. This wave delivers: (1) retrieval inspection point that intercepts retrieved chunks before LLM prompt assembly via the Content-Type Dispatcher, (2) Retrieval Policy Engine with 4 rules â€” classification_clearance, entity_type_restriction, cross_app_isolation, business_unit_isolation, (3) per-chunk policy evaluation (DENY wins on any rule failure), (4) retrieval-time re-detection and anonymization of entities in allowed chunks, (5) RAG token restoration in LLM responses, and (6) policy audit events and Prometheus metrics.

Purpose: Vector stores are typically "read all or read nothing" â€” the Retrieval Policy Engine adds granular per-chunk access control that prevents cross-department leakage, permission bypass, and vector store oversharing.

Output: Retrieved chunks filtered through 4 policy rules; allowed chunks re-inspected and anonymized; LLM response tokens restored.
</objective>

       - `PolicyDecision` dataclass:

       - `RetrievalPolicyEngine` class:
         - Constructor takes `config: dict`:
           - `config["retrieval_policy"]["rules"]` â€” rule enable/disable and parameters.
         - `async evaluate(chunk_metadata: ChunkMetadata, user_context: UserContext) -> PolicyDecision`:
           - Evaluates all enabled rules in order. First DENY â†’ immediate rejection.
           - Returns decision with all matched rules.
           - **RULE-001 (classification_clearance)**:
             - `IF chunk_metadata.classification_level > user_context.clearance_level THEN DENY`.
             - Levels ordered: Public(0) < Internal(1) < Confidential(2) < Restricted(3) < Highly Restricted(4).
           - **RULE-002 (entity_type_restriction)**:
             - `IF user_context.roles does NOT include any role covering chunk_metadata.entity_types_present THEN DENY`.
             - Role mapping: `{"hr": ["ssn", "salary", "employee_id"], "legal": ["case_number", "legal_entity"], "engineering": ["api_key", "source_code"]}`.
           - **RULE-003 (cross_app_isolation)**:
             - `IF chunk_metadata.source_app_id NOT IN user_context.app_ids THEN DENY`.
           - **RULE-004 (business_unit_isolation)**:
             - `IF chunk_metadata.business_unit != user_context.business_unit AND chunk_metadata.classification_level >= Confidential THEN DENY`.
           - **Default**: ALLOW if no rule denies.
           - All missing metadata â†’ treat as Public/empty roles â†’ may still pass.
         - `async evaluate_batch(chunks: list[tuple[ChunkMetadata, str]], user_context: UserContext) -> list[tuple[int, PolicyDecision]]`:
           - Evaluate all chunks, return list of (chunk_index, decision).
         - `load_config(config: dict) -> None`:
           - Loads rule configuration from YAML.
           - Each rule has: `enabled: bool`, `priority: int`.

       - `RAGRetrievalService` class:
         - Constructor takes `policy_engine: RetrievalPolicyEngine`, `detection_engine`, `tokenization_engine`, `vector_connector: VectorStoreConnector`, `cache_manager`, `audit_logger`.
         - `async process_query(query: str, user_context: UserContext, session_id: str, top_k: int = 10) -> dict`:
           1. **Embed query**: `query_embedding = embed(query)` (via configured embedding model).
           2. **Search**: `results = vector_connector.search(query_embedding, top_k=top_k * 2)`.

           6. **Re-detect**: Run Detection Engine on allowed chunk text.
           7. **Re-anonymize**: Run Tokenization Engine on any newly detected entities.

              - New entities: tokenize with new tokens.
           8. **Assemble prompt**: Inject filtered, anonymized chunks into LLM prompt context.
           9. **Audit**: Emit `rag_chunk_filtered` per denied chunk. Emit retrieval metrics.
           10. Return assembled prompt context + chunk metadata list.
