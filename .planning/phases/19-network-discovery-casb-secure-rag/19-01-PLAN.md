<objective>
Build the Shadow AI Discovery pipeline for Phase 19 — passively identifies unauthorized AI service usage by analyzing DNS logs and proxy traffic. This wave delivers: (1) DNS log parser supporting syslog, JSON, and raw-text formats, (2) AI hostname/IP signature database covering 15+ AI providers with wildcard domain matching, (3) hostname matcher that matches parsed queries against signatures using exact/wildcard/CIDR matching, (4) proxy traffic parser supporting Squid, Zscaler, and Palo Alto log formats, (5) usage analyzer computing request patterns per discovered service, (6) shadow AI event generation with configurable webhook alerting, and (7) dedup merge cross-referencing DNS + proxy sources.

Purpose: Without Shadow AI Discovery, enterprises have no visibility into AI services being used outside of IT's knowledge. DNS + proxy analysis provides passive, non-disruptive detection.

Output: Pipeline that ingests DNS and proxy logs, identifies AI service usage, emits shadow_ai_detected events, and outputs merged discovery records.
</objective>

       - `MatchResult` dataclass: `provider: str`, `confidence: float`, `match_type: str`, `signature: ProviderSignature`.
       - `HostnameMatcher` class:
         - Constructor takes `signatures: list[ProviderSignature]`.
         - `match(hostname: str) -> MatchResult | None`:
           - Check exact match first, then wildcard, then multi-level domain.
           - Returns highest confidence match.
         - `match_ip(ip: str) -> MatchResult | None`:
           - Iterates all signature IP ranges, returns first CIDR match.
         - `match_any(hostname: str | None, ip: str | None) -> MatchResult | None`:
           - Tries hostname first (higher confidence), then IP.
         - `refresh_signatures(signatures: list[ProviderSignature]) -> None`:
           - Reloads signatures (for hot-reload support).

       - `ProxyEntry` dataclass: `source_ip: str`, `timestamp: datetime`, `method: str`.
         - `url: str`, `status: int`, `bytes: int`, `user_id: str | None`.
         - `content_type: str | None`, `user_agent: str | None`.
       - `ProxyParser` class:
         - `parse_line(line: str, format: str = "auto") -> ProxyEntry | None`:
           - **Squid format**: timestamp, elapsed, client, status, size, method, url, user, hierarchy, type.
           - **Zscaler format**: CSV with device, datetime, user, department, url, action, reason.
           - **Palo Alto format**: `<timestamp>,<serial>,<type>,<subtype>,<src_ip>,<dest_ip>,<dest_port>,<url>,<user>`.
           - Returns None for unmatched lines (skip, no error).
         - `parse_batch(lines: list[str], format: str = "auto") -> list[ProxyEntry]`:

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-19-01-02 | Information Disclosure | Shadow AI events | mitigate | Events contain metadata only (hostname, IP, service name). No raw DNS query payloads, no request/response bodies. |
| T-19-01-03 | Denial of Service | Log ingestion | mitigate | Batch parsing with configurable max batch size (default 10,000). Streaming parser available for infinite feeds. Parser skips lines exceeding 4KB. |
| T-19-01-04 | Elevation of Privilege | Webhook | mitigate | Webhook URL validated at config time (must be HTTPS). Secrets are not sent in event payloads. |
