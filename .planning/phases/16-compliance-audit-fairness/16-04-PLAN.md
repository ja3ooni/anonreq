  <action>
    Implement eDiscovery export per D-030.

    1. **src/anonreq/models/ediscovery.py**:
       - `ExportFormat` enum: `JSONL`, `PDF`, `EDRM_XML`
       - `EDiscoveryRequest(BaseModel)`: tenant_id (str), date_from (datetime), date_to (datetime), format (ExportFormat), entity_types (list[str] | None=None), case_reference (str | None=None)
       - `EDiscoveryResult(BaseModel)`: format (str), record_count (int), file_size_bytes (int), content_type (str), filename (str), generated_at (datetime), filters (dict)

    2. **src/anonreq/ediscovery/formats.py**:
       - `def to_jsonl(records: list[dict]) -> bytes`:
         - Each record serialized as single JSON line, separated by \n
         - Uses `orjson` or `json.dumps` with default=str for datetimes
       - `async def to_pdf_summary(records: list[dict], metadata: dict) -> bytes`:
         - Generates PDF using `reportlab` or similar
         - Includes: cover page with case info, summary statistics, record listing (session_id, timestamp, entity types), no raw content
       - `def to_edrm_xml(records: list[dict], metadata: dict) -> bytes`:
         - Generates EDRM XML with standard tags:
           - `<DiscoveryRequest>`, `<Case>`, `<Custodian>`, `<Record>`, `<Metadata>`
         - Each lineage record becomes an EDRM `<Record>` entry
         - Uses `xml.etree.ElementTree` or `lxml` for generation

    3. **src/anonreq/ediscovery/export.py**:
       - `class EDiscoveryExporter`:
         - `__init__(self, db: AsyncSession, lineage_tracker: LineageTracker, transparency_service)`
         - `async def export(self, request: EDiscoveryRequest) -> EDiscoveryResult`:
           - Queries lineage records matching tenant_id, date range, optional entity_types
           - Queries transparency records for same scope
           - Merges into export dataset (metadata only, no raw content)
           - Calls format-specific serializer
           - Returns result with file bytes
         - `async def query_export_data(self, request: EDiscoveryRequest) -> list[dict]`:
           - Queries lineage_records + transparency_records with filters
           - Returns list of metadata-only dicts
         - `async def estimate_size(self, request: EDiscoveryRequest) -> dict`:
           - Returns estimated record count and file size without generating full export

    4. **Add to governance router**:
       - `POST /v1/admin/ediscovery/export` — create eDiscovery export
         - Body: `{"tenant_id": "...", "date_from": "...", "date_to": "...", "format": "jsonl", "entity_types": [...], "case_reference": "..."}`
         - Returns file download (Content-Disposition: attachment)
         - Content-Type: `application/jsonl`, `application/pdf`, or `application/xml`
       - `GET /v1/admin/ediscovery/export/estimate` — estimate export size

    5. **tests/test_ediscovery.py**:
       - Test JSONL export format validity
       - Test PDF generation (valid PDF header `%PDF-`)
       - Test EDRM XML schema validity (well-formed XML, correct tags)
       - Test filter application
       - Test metadata-only invariant in export
  </action>
