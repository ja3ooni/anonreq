  <action>
    Implement fairness evaluation pipeline per D-001, D-002.

    1. **src/anonreq/fairness/evaluation.py**:
       - `RECALL_DISPARITY_THRESHOLD = 0.05` (configurable, default from fairness.yaml)
       - `class FairnessEvaluator`:
         - `__init__(self, detection_pipeline, dataset_manager: FairnessDatasetManager, threshold: float = 0.05)`:
           - Stores references to the detection pipeline and dataset manager
         - `async def evaluate_fairness(self, dataset_id: str, version: str, git_sha: str | None = None) -> FairnessEvaluation`:
           - Loads dataset content from MinIO via dataset_manager
           - For each example in dataset:
             - Runs detection pipeline on example text
             - Records whether entity was detected (true positive) or missed (false negative)
           - Groups results by demographic group
           - Computes recall per group: `recall = detected / total`
           - Computes max disparity: `max_disparity = max(recall) - min(recall)` across all groups
           - Builds FairnessResult per entity_type and overall FairnessEvaluation
           - Emits `fairness_evaluation_completed` audit event
           - Returns evaluation
         - `def compute_recall_disparity(self, results: list[DemographicResult]) -> float`:
           - Returns max(recalls) - min(recalls)
         - `def should_fail_build(self, evaluation: FairnessEvaluation) -> bool`:
           - Returns True if any result.max_disparity > threshold
       - CI/CD integration: CLI command `anonreq-fairness-check --dataset <id>`:
         - Runs evaluate_fairness, exits non-zero if should_fail_build

    2. **tests/test_fairness_evaluation.py**:
       - Test recall computation with synthetic dataset
       - Test disparity threshold pass/fail
       - Test per-entity-type results
       - Test audit event emission
  </action>
