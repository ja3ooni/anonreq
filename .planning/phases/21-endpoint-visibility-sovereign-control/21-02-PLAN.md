<objective>
Implement the voice channel connectors and local STT engine for the voice/meeting AI protection pipeline. This wave delivers: SIP trunk proxy, WebRTC media inspection, WebSocket streaming, and gRPC bidirectional stream connectors — all feeding audio into a local self-hosted Whisper STT engine for streaming transcription with overlapping chunk assembly.

Purpose: Without voice connectors and STT, the voice pipeline cannot ingest audio streams. This wave establishes the ingestion layer that Wave 3 (audio sanitization) processes.

Output: All four voice connectors operational with local Whisper STT producing streaming transcripts in an in-memory buffer. Content-Type Dispatcher extended with voice_stream content type.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Voice connector ↔ External network | Inbound audio streams from voice bots/meetings over SIP/WebRTC/WS/gRPC |
| STTEngine ↔ GPU/CPU memory | Model inference within process memory — no network egress |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-02-01 | Information Disclosure | STT audio transcription | mitigate | Local-only inference per Req 50 AC-2. No audio transmitted externally. In-memory only, no disk writes. |
| T-21-02-02 | Tampering | SIP connector message forking | mitigate | SIP message inspection only; no modification of routing headers. |
| T-21-02-03 | Denial of Service | Connector resource exhaustion | mitigate | Configurable timeout (30s default). Connection limits per connector type. |
| T-21-02-04 | Information Disclosure | GPU memory contents | accept | GPU memory isolation is platform responsibility (NVIDIA MPS/ MIG for multi-tenant). |
</threat_model>
