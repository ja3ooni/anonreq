<objective>
Implement the transparent proxy layer with TLS interception and the deployment topology abstraction for all four supported modes (reverse/transparent/virtual/physical). This wave delivers the network interception foundation that all Phase 21 capabilities build on — the ability to intercept, decrypt, inspect, and re-encrypt AI API traffic at the network level without application configuration changes.

Purpose: Without transparent proxy, all Phase 21 features (voice, MCP, AI firewall) apply only to application-configured proxy scenarios. Transparent proxy is the foundation of the sovereign AI control plane.

Output: A working transparent proxy that intercepts requests to api.openai.com (and other AI API endpoints), decrypts via dynamic TLS certs, classifies AI vs non-AI traffic, and forwards through the Content-Type Dispatcher. All deployment modes selectable via DEPLOYMENT_MODE env var.
</objective>

       - `class TLSInterceptor`:
         - `__init__(self, ca_cert_path: str, ca_key_path: str)`: load enterprise CA cert and key
         - `async def generate_cert(self, domain: str) -> ssl.SSLContext`:
           - Generate on-the-fly X.509 cert for domain, signed by enterprise CA
           - SAN includes the domain name
           - Valid for configurable duration (default 24h)
           - Use `cryptography` library for X.509 cert generation
         - `def get_ca_cert_pem(self) -> str`: return CA cert for client trust distribution
         - `async def health_check(self) -> bool`: verify CA cert and key are loadable

       Certificate generation uses:
       ```python
       from cryptography import x509
       from cryptography.x509.oid import NameOID
       from cryptography.hazmat.primitives import hashes, serialization
       from cryptography.hazmat.primitives.asymmetric import rsa
       import datetime

       def _generate_cert(self, domain: str, ca_cert: x509.Certificate, ca_key) -> tuple[x509.Certificate, rsa.RSAPrivateKey]:
           key = rsa.generate_private_key(public_exponent=65537, key_size=2048)
           cert = x509.CertificateBuilder()...
           # SAN: domain, TTL: 24h, signed by CA key
           return cert, key
       ```

    2. **src/anonreq/proxy/transparent_proxy.py**:
       - `class TransparentProxy`:
         - `__init__(self, tls_interceptor: TLSInterceptor, traffic_detector: AITrafficDetector, content_dispatcher, fail_open: bool = False)`
         - `async def start(self, host: str = "0.0.0.0", port: int = 8443)`: Start TCP listener
         - `async def stop(self)`: Graceful shutdown
         - `async def _handle_connection(self, reader, writer)`:
           1. Parse TLS ClientHello to extract SNI (domain name)
           2. Check cert pinning via CertPinningDetector
           3. Classify traffic (AI vs non-AI)

           7. Re-encrypt response and send back to client
         - `async def _forward_untouched(self, reader, writer, host, port)`: raw TCP proxy for non-AI pass-through
         - Protocol preservation: copy HTTP headers, Connection, Keep-Alive, timeouts

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client ↔ Transparent Proxy | Unencrypted traffic after TLS interception inside trust boundary |
| Transparent Proxy ↔ Upstream | Re-encrypted TLS 1.3 outbound to AI provider |
| CA cert/key storage | Enterprise CA private key on filesystem or HSM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-01-01 | Information Disclosure | TLS interception — decrypted traffic in memory | mitigate | Decrypted data held in memory only during pipeline processing. No persistence. Outbound re-encrypted with TLS 1.3. |
| T-21-01-02 | Tampering | Dynamic cert generation | mitigate | Certs signed by enterprise CA, limited TTL (24h). No wildcard certs. |
| T-21-01-03 | Spoofing | Non-AI traffic classification | mitigate | Fail-closed default blocks unclassifiable traffic (HTTP 451). Multi-factor classification (hostname + path pattern). |
| T-21-01-04 | Information Disclosure | CA private key | mitigate | Key stored with restricted filesystem permissions. Optional HSM-backed storage for physical appliance. |
| T-21-01-05 | Denial of Service | Transparent proxy listener | mitigate | Connection limits, timeout config, graceful shutdown. |
</threat_model>
