**1. src/anonreq/middleware/tenant.py:** Create `TenantContextMiddleware(BaseHTTPMiddleware)` following the exact pattern of ClassificationMiddleware in src/anonreq/middleware/classification.py.

Constructor: `__init__(self, app: ASGIApp, tenant_registry: TenantRegistry) -> None`. Store `self._registry = tenant_registry`.

Skip paths: `_SKIP_PATHS = {"/health", "/health/ready", "/metrics", "/"}` — same as PolicyMiddleware.

dispatch method:
- If path is in _SKIP_PATHS or does not start with "/v1/", call `await call_next(request)` and return.
- Extract `tenant_id = request.headers.get("X-AnonReq-Tenant-ID")`.
- If not tenant_id (empty string or missing): return `JSONResponse(status_code=400, content={"error": "missing_tenant", "message": "X-AnonReq-Tenant-ID header required"})` per D-01. No implicit default tenant.
- Look up `profile = self._registry.get(tenant_id)`.
- If profile is None: return `JSONResponse(status_code=400, content={"error": "invalid_tenant", "message": f"Unknown tenant: {tenant_id}"})`.
- If profile is not None and `profile.enabled is False`: return `JSONResponse(status_code=403, content={"error": "tenant_disabled"})` per D-04.
- Set `request.state.tenant_id = tenant_id` and `request.state.tenant_profile = profile` per D-03.
- Bind to structlog contextvars: `structlog.contextvars.bind_contextvars(tenant_id=tenant_id)` per D-10.
- Try/finally block: `response = await call_next(request)` in try, `structlog.contextvars.unbind_contextvars("tenant_id")` in finally.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Gateway | Untrusted X-AnonReq-Tenant-ID header crosses here; must be validated against registry |
| Gateway → Cache | Tenant-scoped key prefix isolates data; validated tenant_id prevents cross-tenant access |
| Middleware → Pipeline | request.state.tenant_id carries validated context; pipeline stages never read headers directly |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-31-01 | Tampering | Tenant header spoofing | high | mitigate | TenantContextMiddleware validates header against TenantRegistry; unknown tenants rejected with HTTP 400 |
| T-31-02 | Information Disclosure | Cross-tenant cache bleed | critical | mitigate | Tenant-prefixed cache keys (anonreq:{tenant_id}:{session_id}); validated tenant_id from middleware, not client input |
| T-31-03 | Elevation of Privilege | Disabled tenant access | high | mitigate | HTTP 403 returned for enabled=false tenants; distinct from 400 for missing/invalid per D-04 |
| T-31-04 | Denial of Service | Registry lookup DoS | low | accept | In-memory dict lookup is O(1); no external calls during request path |
| T-31-SC | Tampering | PyYAML safe_load | low | mitigate | Uses yaml.safe_load() for config/tenants.yaml; prevents code injection from malicious YAML |
</threat_model>
