**2. src/anonreq/kms/base.py:** Define `KMSClient` ABC with two abstract async methods:
- `async def encrypt(self, tenant_id: str, plaintext: bytes) -> bytes` — Encrypt plaintext using tenant-specific key. Returns ciphertext bytes (includes nonce/tag prepended).
- `async def decrypt(self, tenant_id: str, ciphertext: bytes) -> bytes` — Decrypt ciphertext using tenant-specific key. Returns plaintext bytes. Raises on tampering (AEAD verification).

Docstrings must reference D-07, D-08.

**3. src/anonreq/kms/cache.py:** Define `InMemoryKeyCache` dataclass and class per D-09:
- `CachedKey` dataclass with `data_key: bytes` and `expires_at: float`.
- `InMemoryKeyCache.__init__(self, ttl_seconds: int = 300, max_entries: int = 1000)`.
- `_cache: dict[str, CachedKey]` private store.
- `async def get_or_derive(self, tenant_id: str, master_key: bytes) -> bytes`: Check cache first (key by tenant_id). On miss or TTL expiry, derive a 256-bit data key using `cryptography.hazmat.primitives.kdf.hkdf.HKDF` with algorithm SHA256, length=32, salt=None, info=f"anonreq-tenant-{tenant_id}".encode(). Evict oldest entry if cache at capacity. Store with `expires_at = time.monotonic() + self._ttl`. Return the derived key.
- Use `cryptography.hazmat.primitives.kdf.hkdf.HKDF` and `cryptography.hazmat.primitives.hashes.SHA256` from the already-installed `cryptography` library.
- `async def invalidate(self, tenant_id: str) -> None` — Remove cached key for a tenant (for rotation signal).
- `async def clear(self) -> None` — Clear all cached keys (for testing).

**4. src/anonreq/kms/local.py:** Define `LocalAES256GCM(KMSClient)` per D-07:
- Constructor: `__init__(self, master_key: bytes, key_cache: InMemoryKeyCache)`.
- `encrypt(tenant_id, plaintext)`: Get data_key from `self._cache.get_or_derive(tenant_id, self._master_key)`. Generate random 12-byte nonce via `os.urandom(12)`. Create `AESGCM(data_key)` from `cryptography.hazmat.primitives.ciphers.aead.AESGCM`. Encrypt: `ciphertext_with_tag = aesgcm.encrypt(nonce, plaintext, None)`. Return `nonce + ciphertext_with_tag` (prepend nonce for decrypt).
- `decrypt(tenant_id, ciphertext)`: Split first 12 bytes as nonce, remainder as ciphertext_with_tag. Get data_key from cache. Create AESGCM, decrypt. AEAD verification happens automatically — raises `InvalidTag` on tampering.
- Add a classmethod `generate_master_key() -> bytes` that returns `os.urandom(32)` for generating a fresh master key.

Modify `store_mapping`: Before calling `pipe.hset(key, mapping=mapping)`, if `self._kms is not None`, encrypt each value in the mapping dict. The encrypted mapping becomes: `{token: base64.b64encode(await self._kms.encrypt(tenant_id, value.encode())).decode() for token, value in mapping.items()}`. Use `base64.b64encode` for safe Valkey storage (Valkey stores strings). Store the encrypted values instead of plaintext.

Modify `get_mapping`: After `hgetall`, if `self._kms is not None`, decrypt each value: `{token: (await self._kms.decrypt(tenant_id, base64.b64decode(ciphertext))).decode() for token, ciphertext in raw.items()}`. Return decrypted mapping.

CRITICAL FAIL-SECURE: If decryption fails (e.g., tampered data, wrong key), catch the exception and raise `DependencyUnavailableError(dependency="kms")` — this follows the existing fail-secure pattern for cache failures and prevents partial/corrupt data from reaching the pipeline. The existing retry decorator on `_execute_with_retry` already converts retryable errors to DependencyUnavailableError.

The `_kms` attribute is stored as `self._kms: KMSClient | None` in `__init__`. When None, store/get operate on plaintext (backward compatible).

**2. src/anonreq/main.py:** In the lifespan function, after creating the TenantRegistry and CacheManager, create the KMS backend based on settings.KMS_BACKEND and attach to CacheManager:

from anonreq.kms.local import LocalAES256GCM
from anonreq.kms.cache import InMemoryKeyCache

kms_backend = settings.KMS_BACKEND.strip().lower()
if kms_backend == "local":
    master_key = os.environ.get("ANONREQ_KMS_MASTER_KEY", "")
    if not master_key:
        # Generate ephemeral master key for dev/testing (not for production)
        master_key = LocalAES256GCM.generate_master_key()
    key_cache = InMemoryKeyCache(ttl_seconds=settings.CACHE_TTL_SECONDS)
    kms_client = LocalAES256GCM(master_key=master_key, key_cache=key_cache)
    cache_manager._kms = kms_client

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CacheManager → Valkey | Ciphertext crosses this boundary; plaintext never touches Valkey per D-08 |
| KMSClient → InMemoryKeyCache | Data keys cached in-memory with bounded TTL per D-09 |
| Master Key → Data Key | HKDF derivation isolates per-tenant; master key compromise exposes all tenant data |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-31-05 | Information Disclosure | Valkey data dump exposure | critical | mitigate | Encrypt-at-storage-layer per D-08; ciphertext stored in Valkey |
| T-31-06 | Tampering | Ciphertext tampering in Valkey | high | mitigate | AES-256-GCM AEAD detects tampering; decryption raises InvalidTag → DependencyUnavailableError |
| T-31-07 | Elevation of Privilege | Master key compromise | critical | accept | Per-tenant data keys via HKDF; scope limited to cache data; operational concern for key management |
| T-31-08 | Denial of Service | KMS call latency | medium | mitigate | InMemoryKeyCache with bounded TTL per D-09 avoids KMS call per request |
| T-31-09 | Information Disclosure | Key cache eviction exposes keys | low | accept | Evicted keys are not persisted; in-memory only; TTL-based expiry |
</threat_model>
