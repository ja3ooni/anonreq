# Plan 21-06: Air-Gapped Deployment Mode

**Objective**: Fully disconnected operation — all provider traffic routed to local models, no external network required.

**Depends on**: 21-05 (hybrid selector), 21-03 (local model router)

## Tasks

1. `src/anonreq/endpoint/airgap.py` — `AirGapMode` lifecycle manager
2. Detection: DNS resolution failure for external providers → auto air-gap
3. Configuration: `mode: airgap` in endpoint.yaml forces disconnected mode
4. All external provider routes blocked at proxy level
5. Local model fallback guaranteed for every classification level
6. Health check: `GET /v1/endpoint/airgap/status` with connectivity probes
7. `airgap_mode_entered`, `airgap_mode_exited`, `airgap_blocked` audit events
8. Prometheus: `airgap_active` gauge
9. 15+ tests
