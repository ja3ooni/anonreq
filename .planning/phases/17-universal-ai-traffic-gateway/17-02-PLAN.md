<objective>
Build the AI traffic detection layer and MCP protocol inspection for the Universal AI Traffic Gateway. This wave delivers: (1) PAC file generation at `GET /v1/proxy.pac` with all known AI provider domains, (2) hostname allowlist for AI provider detection with wildcard matching and IP range matching, (3) flow analysis heuristics to identify AI API traffic patterns in requests that bypass proxy configuration, (4) block-all-unintercepted-AI enforcement as a Phase 8 PDP #2 policy rule, and (5) full MCP protocol parser and inspector integrated with the Content-Type Dispatcher.

Purpose: The gateway must detect all AI traffic regardless of how it enters the network — explicitly proxied (PAC), known AI endpoints (allowlist), or pattern-matched (flow analysis). MCP protocol inspection ensures agent/tool traffic is equally visible to policy enforcement.

Output: Gateway detects AI traffic across 3 layers, serves auto-config PAC, blocks bypassing traffic, and inspects MCP protocol messages.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → PAC endpoint | PAC file served to all clients — domain list is public info |
| Traffic bypass analysis | Flow analysis inspects request metadata (no raw body stored) |
| MCP message content | MCP parser reads tool call arguments and tool results |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-02-01 | Information Disclosure | PAC file | accept | PAC file lists provider domains only (public info). No credentials, no internal topology. |
| T-17-02-02 | Tampering | Flow analysis | mitigate | Flow analyzer is read-only — inspects, does not modify. All results ephemeral. |
| T-17-02-03 | Information Disclosure | MCP message inspection | mitigate | MCP parser processes tool call arguments — raw arguments visible to gateway only during policy evaluation. Arguments anonymized per Phase 9 before forwarding. |
| T-17-02-04 | Denial of Service | Flow analyzer CPU usage | mitigate | Analysis limited to first 4KB of body. Timeout per request: 50ms. All analysis is optional (skip on resource pressure). |
| T-17-02-05 | Bypass | block-all-unintercepted-AI | mitigate | Three independent detection layers — bypassing one still triggers another. `X-AnonReq-Proxy` header verification prevents spoofed proxy-origin claims. |
</threat_model>
