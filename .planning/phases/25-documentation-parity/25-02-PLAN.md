    ---

    **Translation rules (per D2, D7):**
    - Translate ALL prose content to the target language. Technical documentation should read naturally in the target language.
    - Code blocks (bash, python, yaml, json, dockerfile, etc.) remain in English — never translate variable names, command arguments, file paths, or config values (per D7).

    1. **Update GLOSSARY:** For each of ~20 glossary terms, add the translation in the target language column. If the source English doc uses a term not yet in the glossary, add a new row.

    - French: Use "vous" formal tone for consistency with enterprise documentation. Translate "Getting Started" as "Pour commencer".
