<objective>
Implement the fail-secure core infrastructure: global exception handler with OpenAI-compatible error envelope, structured audit logging with strict field allowlist, health endpoint, and pre-flight startup checks. This is the heart of the Phase 1 leak-free guarantee.

Purpose: Every error — whether from Python code, FastAPI, or a crashed dependency — must produce a safe HTTP 500 with zero information leakage. Logs must never contain raw values. The gateway must refuse to start if dependencies are unhealthy. Nothing that touches request data can ship until these guarantees are proved.

Output: A working FastAPI application where `GET /health` returns component status, pre-flight checks block startup on dependency failure, error responses are safe, and logs are structured JSON with only allowlisted fields.
</objective>

       - `AnonReqError(Exception)` — base class with `message`, `error_type`, `status_code`, `code`, `request_id` fields.
       - `DependencyUnavailableError(dependency: str, request_id: str | None = None)` — returns 503, type `service_unavailable`, code `dependency_unavailable`. Message format: `"{dependency} unavailable"`.

         - Close connection.
         - Return `True` on success, `False` on failure.

       - `async def check_presidio(url: str) -> bool`:

         - Return `True` if status < 500, `False` otherwise.
       - `async def run_startup_checks(settings) -> None`:
         - Call `check_valkey(settings.VALKEY_URL)` and `check_presidio(settings.PRESIDIO_URL)`.
         - If either fails, raise `DependencyUnavailableError(dependency_name)` with the name of the failing dependency.
         - Log each check result via structlog.
       - Log all check results with component="startup_checks" before raising.

             "status": "healthy",
             "version": "0.1.0",

                 raise  # uvicorn exits — fail-secure per FAIL-04
             log.info("lifespan", event="Pre-flight checks passed, accepting traffic")
             yield
             log.info("lifespan", event="Shutting down")

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Exception handler → Response | Any unhandled exception crosses from internals to client response |
| Logger → stdout | Log data crosses from application to Docker log collector |
| Pre-flight checks → Startup | Dependency status gates service availability |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Information Disclosure | global_exception_handler | mitigate | Every exception returns generic error envelope per D-02. No stack trace, no request body, no internal URLs, no env var values, no dependency URLs in response. `--no-server-header` prevents version leak per RESEARCH Pitfall 2. |
| T-01-03-02 | Information Disclosure | logging_config allowlist | mitigate | Field allowlist processor drops non-allowlisted keys per D-05. Nested-dict limitation documented (acceptable in Phase 1 — no request data flows yet). Recursive deep check added in Phase 2 when request data flows. |
| T-01-03-03 | Denial of Service | startup_checks | mitigate | `socket_connect_timeout=3` prevents hung checks. `httpx timeout=5` for Presidio check. Fast fail on first dependency failure. |
| T-01-03-04 | Elevation of Privilege | startup_checks network calls | accept | Pre-flight makes outbound TCP connections to Valkey and Presidio. Both are on internal Docker network. Risk of SSRF is minimal — URLs come from config, not user input. |
| T-01-03-SC | Tampering | structlog/redis/httpx packages | mitigate | All packages verified in RESEARCH.md § Package Legitimacy Audit. [SUS] verdicts overridden — well-known, reputable libraries. |
</threat_model>
