  <name>Task 1: Create multi-stage Dockerfile</name>
  <files>
    Dockerfile,
    .dockerignore
  </files>
  <action>
    Create a multi-stage Dockerfile at the project root per DOCK-02.

    **Builder stage** (`python:3.12-slim`):
    - `WORKDIR /app`
    - `COPY requirements.txt .` (install deps before source for layer caching)
    - `RUN pip install --no-cache-dir -r requirements.txt`
    - `COPY pyproject.toml .`
    - `COPY src/ src/`
    - `COPY config/ config/`
    - `RUN pip install --no-cache-dir -e .`

    **Runtime stage** (`python:3.12-slim`):
    - `WORKDIR /app`
    - Copy virtual env or installed packages from builder:
      - `COPY --from=builder /usr/local/lib/python3.12/site-packages/ /usr/local/lib/python3.12/site-packages/`
      - `COPY --from=builder /usr/local/bin/ /usr/local/bin/`
    - `COPY src/ src/`
    - `COPY config/ config/`
    - `EXPOSE 8080`
    - `HEALTHCHECK --interval=10s --timeout=5s --retries=3 --start-period=5s CMD curl -f http://localhost:8080/health || exit 1`
    - `CMD ["uvicorn", "anonreq.main:app", "--host", "0.0.0.0", "--port", "8080", "--no-server-header"]`
    - Note: `--no-server-header` per RESEARCH Pitfall 2 (prevents Uvicorn version leak).

    **.dockerignore**:
    - `.git/`, `.venv/`, `__pycache__/`, `*.pyc`, `.env`, `.env.example`, `tests/`, `requirements-dev.txt`, `.dockerignore`, `*.md`

    Ensure final image stays under 2GB per DOCK-02.
  </action>

    - container_name: `anonreq-valkey`
    - command: `["valkey-server", "--save", "", "--appendonly", "no"]` (persistence disabled per DOCK-03)

    - No port exposure (internal only)

    **presidio-analyzer** (image: `mcr.microsoft.com/presidio-analyzer:latest`):
    - container_name: `anonreq-presidio`
    - healthcheck: `curl -f http://localhost:5001/health` (or fallback to TCP check), interval 15s, timeout 5s, retries 10, start_period 30s
    - networks: `anonreq-net`
    - No port exposure (internal only)

    **anonreq** (build: `.`):
    - container_name: `anonreq-gateway`
    - ports: `"8080:8080"`
    - environment:
      - `ANONREQ_API_KEY=${ANONREQ_API_KEY:?err}` â€” fails if unset per DOCK-07
      - `ANONREQ_VALKEY_URL=redis://valkey:6379/0`
      - `ANONREQ_PRESIDIO_URL=http://presidio-analyzer:5001`
      - `ANONREQ_LOG_LEVEL=${ANONREQ_LOG_LEVEL:-INFO}`
      - `ANONREQ_HOST=0.0.0.0`
      - `ANONREQ_PORT=8080`
    - depends_on with `condition: service_healthy` for both valkey and presidio per DOCK-04
    - healthcheck: `curl -f http://localhost:8080/health`, interval 10s, timeout 5s, retries 3, start_period 5s
    - restart: `unless-stopped`
    - networks: `anonreq-net`

    **networks**:
    - `anonreq-net`: driver `bridge` â€” isolates all 3 services from host network per DOCK-03
