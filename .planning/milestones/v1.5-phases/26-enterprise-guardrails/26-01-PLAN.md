  <action>
    Create `src/anonreq/detection/recognizers/enterprise.py` following the same pattern as `mnpi.py`. This file contains four recognizer classes, each producing detection dicts with keys `entity_type`, `start`, `end`, `score`, `source`.

    Per D-03 (explicit naming): each recognizer name includes AnonReq_ prefix to avoid dict-key collision in Presidio.

    **1. AnonReq_APIKeyRecognizer**
    - Entity type: `ENTERPRISE_API_KEY`
    - Regex: `\b(sk-[a-zA-Z0-9]{20,}|pk-[a-zA-Z0-9]{20,}|sk-proj-[a-zA-Z0-9]{20,})\b`
    - Covers OpenAI-style secret keys, project keys, and other common API key formats
    - Confidence: 0.85 (configurable)
    - Source: `"enterprise"`

    **2. AnonReq_AWSAccessKeyRecognizer**
    - Entity type: `ENTERPRISE_AWS_KEY`
    - Regex: `\b(AKIA[0-9A-Z]{16})\b`
    - Confidence: 0.90 (configurable)
    - Source: `"enterprise"`

    **3. AnonReq_GitHubTokenRecognizer**
    - Entity type: `ENTERPRISE_GITHUB_TOKEN`
    - Regex: `\b(ghp_[a-zA-Z0-9]{36}|ghs_[a-zA-Z0-9]{36}|gho_[a-zA-Z0-9]{36}|ghu_[a-zA-Z0-9]{36}|ghr_[a-zA-Z0-9]{36}|ghb_[a-zA-Z0-9]{36})\b`
    - Covers GitHub personal access tokens (ghp_), OAuth (ghs_), org (gho_), user-to-server (ghu_), refresh (ghr_), and installation (ghb_) tokens
    - Confidence: 0.90 (configurable)
    - Source: `"enterprise"`

    **4. AnonReq_InternalHostnameRecognizer**
    - Entity type: `ENTERPRISE_INTERNAL_HOST`
    - FQDN matching against configurable internal domain list (e.g. `.internal`, `.corp.`, `.local`, `*.company.com`)
    - Uses regex built from the configured domain list: `\b([a-zA-Z0-9]([a-zA-Z0-9-]*[a-zA-Z0-9])?\.)+({domain_pattern})\b`
    - The domain list is read from YAML config (not hardcoded)
    - Confidence: 0.80 (configurable)
    - Source: `"enterprise"`

    Each recognizer has:
    - Config dataclass (following MNPIConfig pattern) holding pattern, confidence, enabled flag
    - An `analyze(text, tenant_id=None)` method returning `list[dict[str, Any]]`
    - Per D-01: results compatible with the existing detection pipeline dict format
    - Overlapping match deduplication following the MNPIRecognizer._deduplicate pattern
    - Recognizer name exposed as a `name` property/attribute using the D-03 names (AnonReq_APIKeyRecognizer, AnonReq_AWSAccessKeyRecognizer, AnonReq_GitHubTokenRecognizer, AnonReq_InternalHostnameRecognizer)

    At module level, export an `EnterpriseRecognizerConfig` dataclass with fields:
    - `enabled: bool` (default True) — per recognizer enable/disable
    - `confidence: float` — per recognizer confidence override
    - `patterns: list[str]` — optional custom regex patterns
    - `internal_domains: list[str]` — for the hostname recognizer

    Add a `create_enterprise_bundle(config_path="config/recognizers.yaml")` factory function (mirroring `create_mnpi_bundle`) that:
    - Reads YAML config
    - Creates enabled recognizers based on config
    - Returns a dict `{"api_key": AnonReqAPIKeyRecognizer, "aws_access_key": ..., "github_token": ..., "internal_hostname": ...}` 
    - Gracefully returns empty dict if config not found (log warning only, per existing graceful degradation pattern)

    Per D-02: configuration keys from YAML use underscores, matching Pythonic naming style for the dataclass fields.

    Use `from __future__ import annotations`, proper type hints, and comprehensive docstrings matching the mnpi.py style.
