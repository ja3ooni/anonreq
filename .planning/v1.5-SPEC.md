## Overview

hygiene (P1), customer-facing Trust Center and documentation parity (P2), and enterprise

## Phase 1 — Engineering Hygiene

**Goal:** Fix foundational gaps before adding features. No new functionality.

### 1.1 CI/CD Test Workflow

- Create `.github/workflows/test.yml`:
  - Trigger: push to `main`, pull requests targeting `main`
  - Matrix: Python 3.12 only (matching project constraint)
  - Steps: checkout → install deps with `uv sync --group dev` → `uv run pytest tests/ -x -v`
    (excluding `-m load` tests, which are manual-trigger)
  - Add coverage reporting step (threshold ≥ 70%)
  - Mark load tests with `if: github.event_name == 'workflow_dispatch'`

### 1.2 Code Quality Enforcement

- Add `ruff` and `mypy` to `pyproject.toml` `[project.optional-dependencies] dev`
- Create `ruff.toml`:
  - `target-version = "py312"`
  - `line-length = 100`
  - Select: `E`, `F`, `I`, `N`, `W`, `UP`, `B`, `SIM`, `ARG`, `PT`, `RUF`
  - Ignore: select rules that conflict with existing patterns (document in ruff.toml)
- Create `mypy.ini`:
  - `python_version = 3.12`
  - `strict = true` with per-module overrides for known-untyped deps (fastapi, pydantic, etc.)
- Add CI step: `uv run ruff check src/ tests/` and `uv run mypy src/`
- **Auto-fix sweep:** Run `ruff check --fix src/ tests/` and review all changes. For
  mypy violations, fix iteratively. Commit violation fixes per category.
- Add `[tool.ruff]` inline to `pyproject.toml` as single source of truth (remove separate
  `ruff.toml` after writing inline).

### 1.3 Secure Docker Defaults

- Remove host port bindings from all services except `anonreq` (gateway):
  - `postgres`: remove `ports: ["5432:5432"]`
  - `minio`: remove `ports: ["9000:9000", "9001:9001"]`
  - `prometheus`: remove `ports: ["9090:9090"]`
  - `grafana`: remove `ports: ["3000:3000"]`
- Disable Grafana anonymous auth:
  - `GF_AUTH_ANONYMOUS_ENABLED=false`
  - `GF_AUTH_ANONYMOUS_ORG_ROLE=Viewer` → remove
- Move observability ports behind the `observability` profile (already in profile for some;
  ensure all non-gateway services are consistent).
- Add comment documenting that production deployments should use a reverse proxy for TLS
  termination.

### 1.4 Verification

- `uv run pytest tests/ -x` passes
- `ruff check src/ tests/` passes
- `mypy src/` passes (no `--ignore-missing-imports` hacks)
- `docker compose up` starts with gateway on `8080` only (no other host ports)

## Phase 2 — Trust Center

**Goal:** Public-facing compliance evidence portal matching Vanta Trust Center baseline.

### 2.1 Package Scaffold

```
src/anonreq/trust_center/
├── __init__.py
├── router.py       # FastAPI APIRouter
├── schemas.py      # Pydantic response models
├── service.py      # Business logic (read from SLO engine, compliance registry, metrics)
└── config.py       # TrustCenterSettings (pydantic-settings)
```

Config toggle: `trust_center.enabled` (default `false`, boolean). When disabled, all
routes return 404.

### 2.2 Public Endpoints

All under `/v1/trust/`, no auth required, rate-limited to 60 RPM per IP.

| Endpoint | Response | Source |
|---|---|---|
| `GET /v1/trust/status` | Aggregated SLO compliance summary | SLO engine |
| `GET /v1/trust/compliance` | Supported frameworks + presets list | Compliance registry |
| `GET /v1/trust/metrics` | Aggregate throughput, entity counts, uptime | Prometheus metrics |
| `GET /v1/trust/security` | Security posture summary | config/trust_center.yaml |

All responses are metadata-only. No tenant-level data, no raw values, no session data.

### 2.3 Configuration

`config/trust_center.yaml`:
```yaml
enabled: false
display_name: "AnonReq Trust Center"
contact_email: "security@example.com"
logo_url: ""
supported_frameworks:
  - soc2
  - iso27001
  - gdpr
  - hipaa
feature_summary:
  anonymization: true
  dlp: false       # redact for public if not yet enabled
  firewall: false
```

### 2.4 Integration

- Register `trust_center.router` in `main.py` behind config gate
- Wire `service.py` to:
  - `SLOEngine` for SLO summaries (read-only aggregated)
  - `ComplianceRegistry` for framework/preset listing
  - `prometheus_client` `REGISTRY` for aggregate metrics
- Fail-closed: if dependent service unavailable, return 503

### 2.5 Tests

- Unit: config parsing, schema validation, fail-closed when disabled
- Integration: full response format verification for each endpoint
- Security: rate limiting behavior, no tenant data exposure

## Phase 4 — Enterprise Guardrails

**Goal:** Match NodeShift secret detection, add Vanta-like compliance automation,
implement commercial packaging.

### 4.1 Advanced Secret Detection

Custom Presidio recognizers in `src/anonreq/detection/recognizers/`:

| Recognizer | Pattern | Confidence |
|---|---|---|
| `APIKeyRecognizer` | `sk-...`, `pk-...`, `sk-proj-...` (OpenAI-style) | 0.85 |
| `AWSAccessKeyRecognizer` | `AKIA[0-9A-Z]{16}` | 0.90 |
| `GitHubTokenRecognizer` | `ghp_[a-zA-Z0-9]{36}`, `ghs_[a-zA-Z0-9]{36}` | 0.90 |
| `InternalHostnameRecognizer` | FQDNs matching internal domains from config | 0.80 |

Config: `config/recognizers.yaml` with:
- Enable/disable per recognizer
- Internal domain list for hostname matching
- Confidence thresholds

Integration: register with RecognizerMerger in locale pipeline. Tests with known
valid/invalid patterns.

### 4.2 Continuous Compliance Monitoring

Extend `AuditChainService` with:

- `GET /v1/admin/compliance/evidence?framework=soc2` — structured evidence per framework
  control, sourced from:
  - SLO compliance state
  - Audit chain entries
  - Governance records
  - Incident history
- Scheduled evidence snapshots (configurable cron, default daily) that persist to
  compliance archive (MinIO or filesystem)
- Evidence bundle available as JSON Lines download

Reference: existing `ComplianceRegistry` in `src/anonreq/compliance/` provides the
framework/control mapping — this phase wires live data into it.

### 4.3 Commercial Licensing Enforcement

```
src/anonreq/license/
├── __init__.py
├── models.py        # LicenseKey, FeatureGate, LicenseStatus
├── validator.py     # Key verification (HMAC-SHA256 symmetric)
├── config.py        # LicenseSettings
└── router.py        # GET /v1/admin/license
```

**Mechanism:** HMAC-SHA256 symmetric signing of license payload (org, tier, features,
expiry). Key stored in `ANONREQ_LICENSE_SECRET` env var. Validation at startup and
on each gated feature access. No phone-home required.

**Feature gates:**
| Gate | Blocks | Default |
|---|---|---|
| `trust_center` | Trust Center routes | Free (core) |
| `ai_firewall` | AI Firewall module | Appliance |
| `soc_integration` | SOC/SIEM sinks | Appliance |
| `advanced_detection` | Custom recognizers from 4.1 | Appliance |
| `compliance_monitoring` | Evidence endpoint from 4.2 | Appliance |

**Enforcement pattern:**
```python
async def check_license(gate: FeatureGate) -> bool:
    license = await LicenseValidator.get_status()
    return gate in license.features and not license.expired
```

Used as FastAPI dependency on gated routes. Core anonymization pipeline is NOT gated.

**Tests:**
- Valid license → gate passes
- Expired license → gate blocks with 402
- Missing license key → gate blocks with 402
- Invalid signature → gate blocks with 402
- Core pipeline still works without any license

### 4.4 Integration

- Wire license check as dependency on Trust Center routes (Phase 2)
- Wire license check on SOC integration routes (already exists — add gate)
- Wire license check on custom recognizer loading (only load if licensed)
- Add `GET /v1/admin/license` returning current license status (tier, features, expiry)

## Key Constraints

- **No PII in Trust Center:** Trust Center endpoints return aggregate metadata only.
  No tenant breakdowns, no session data, no raw values.
- **Fail-closed:** If license validator cannot reach config or key is missing, block
  gated features. If Trust Center dependent service is down, return 503.
- **Core remains free:** Anonymization pipeline, detection engine, tokenization,
  restoration, and streaming are never gated. Only Appliance-tier additions are gated.
- **Docker security:** After Phase 1, default `docker compose up` exposes only port
  8080. Observability ports require explicit `--profile observability` flag.
